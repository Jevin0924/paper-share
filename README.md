# 📘 感知线论文分享仓库（paper-share）

## 1. 仓库目标
本仓库用于：
- 跟进 **计算机视觉领域前沿论文**；
- 结合业务场景，评估其 **落地可行性与潜在价值**；
- 组织定期分享、实验验证与成果转化，形成 **技术闭环**。

旨在促进感知团队在检测、人脸属性、分类、识别、跟踪等方向持续创新，
并同步推进量化与轻量化专项的研究与工程落地。

---

## 2. 分享机制

- 📅 **分享周期**：每周二下午  
- 👤 **分享人**：各方向调研负责人  
- 📑 **内容要求**：
  1. 论文核心技术点解析  
  2. 与项目模型对比实验结果  
  3. 可落地性分析与验证建议  
  4. 最终结论与行动项（落地 / 小试 / 观察 / 关闭）

---

## 3. 仓库结构

```bash
perception-paper-share/
├── README.md                 # 仓库说明（本文档）
├── TEMPLATE.md               # 论文分享模版
├── papers/                   # 每篇论文独立目录
│   ├── 20251009_YOLO26/
│   │   ├── README.md         # 按模板填写
│   │   ├── code/             # 若有复现代码
│   │   └── results/          # 实验数据与可视化结果
│   ├── 20251009_DEIM/
│   └── ...
├── summary/                  # 季度总结、选题回顾
│   ├── 2025Q3_summary.md
│   └── 2025Q4_summary.md
└── tools/                    # 指标统计、可视化等通用脚本
```

---

## 4. 提交流程

### 📄 新论文分享
1. 在 `papers/` 目录下创建子目录，命名规则：
   ```bash
   papers/YYYYMMDD_<PaperName>/
   ```
   示例：
   ```bash
   papers/20251009_YOLO26/
   ```

2. 在该目录下填写 `README.md`，使用标准模板：
   ```bash
   TEMPLATE.md
   ```

3. 提交后发起 `Merge Request` 到 `main`，标题格式为：
   ```bash
   [Share] YOLO26: Real-Time Object Detection on Edge
   ```

---

## 5. 分支与命名规范

| 分支名 | 用途 | 示例 |
|--------|------|------|
| `main` | 主分支，存放正式论文分享内容 | - |
| `dev/<user>` | 个人开发或复现分支 | `dev/alice` |
| `feature/<paper>` | 单篇论文复现与验证分支 | `feature/yolo26` |

---

## 6. 论文分享内容要求（概要）

| 模块 | 内容要求 |
|------|-----------|
| **论文简介** | 论文标题、作者、核心贡献、关键技术点、数据集与指标 |
| **结果对比** | 与项目模型在同测试集下的精度、速度、参数量对比 |
| **可落地性分析** | 适配场景、优势与限制、工程实现风险 |
| **实验建议** | 可复现模块与预期收益 |
| **结论与行动项** | 落地决策与负责人、时间节点 |

详见 [`TEMPLATE.md`](./TEMPLATE.md)。

---

## 7. 注意事项
- 所有实验需在 **同一测试集与输入大小** 下进行对比。  
- 优先关注 **轻量化 / 高效模型 / 蒸馏结构** 的论文。  
- 若论文未开源，仅复现核心模块（ablation-style）。  
- 严格遵守数据隐私与安全规范，禁止上传内部或敏感数据。  
- 分享时长建议 **15~20分钟**，重点突出可落地价值。

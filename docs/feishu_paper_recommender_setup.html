每日论文推荐飞书配置手册

按本手册完成飞书多维表格、自建应用、群机器人、本地环境和定时任务配置后，本地 Codex 会每天生成感知算法方向论文推荐，写入飞书多维表格并推送到飞书群。

总览 多维表格 飞书应用 群机器人 本地环境 验证流程 定时任务 排错

1. 配置总览

本地脚本 抓取 arXiv，规则打分，调用本地 codex exec --search 生成摘要。
飞书多维表格 每篇论文一条记录，用于去重、追溯、检索和后续人工标注。
飞书自建应用 通过 APP_ID 和 APP_SECRET 获取 token，写入多维表格。
飞书群机器人 把每日 3-5 篇推荐推送到群，消息中附论文链接和表格链接。

配置完成后的主命令是：

cd /mnt/d/faceid/code/paper-share
python3 tools/paper_recommender/run_daily.py
建议先按本文的验证顺序从 --sample 开始，不要一上来就跑真实推送。

2. 创建飞书多维表格

  1. 在飞书中新建一个多维表格，建议命名为 每日论文推荐追溯库。
  2. 新建一个数据表，建议命名为 论文推荐。
  3. 按下表创建字段。字段名必须和下表一致，除 推荐分 外建议都先用文本字段。
也可以直接导入模板文件创建表格：docs/feishu_bitable_template.xlsx。如果飞书导入 Excel 异常，使用兜底 CSV：docs/feishu_bitable_template.csv。
字段名 推荐字段类型 说明
推荐日期文本脚本运行日期，例如 2026-05-29。
首次发现时间文本第一次写入该论文的 UTC 时间。
最近更新时间文本重复运行时更新。
arXiv ID文本主要去重键，例如 2605.00001。
DOI文本Semantic Scholar 补充到时写入。
标题文本论文标题，也是兜底去重键。
作者文本作者列表，过长会截断。
机构文本当前代码写入 venue 或补充来源信息。
论文链接文本arXiv 页面链接。
代码链接文本Codex 能确认官方代码时写入。
类别文本arXiv 类别，如 cs.CV。
关键词文本命中的打分关键词。
推荐分数字规则打分结果。
推荐等级文本强推荐、可观察、暂不推荐。
一句话结论文本Codex 生成的短结论。
核心贡献文本论文核心贡献。
技术要点文本多条技术点会以换行写入。
实验亮点文本主要实验结果或需重点核验的指标。
落地价值文本对感知业务的参考价值。
风险限制文本复现、部署、数据或实验风险。
建议动作文本小试、观察、关闭。
是否已推送文本群消息发送成功后写入 是。
不要把 推荐日期、首次发现时间、最近更新时间 先建成日期字段。当前脚本写入的是字符串，日期字段可能要求时间戳，容易导致 API 写入失败。

获取多维表格参数

打开多维表格后，从 URL 中识别：

https://xxx.feishu.cn/base/appXXXXXXXX?table=tblXXXXXXXX&view=vewXXXXXXXX
环境变量值
FEISHU_BITABLE_APP_TOKENappXXXXXXXX
FEISHU_BITABLE_WIKI_TOKEN如果链接是 feishu.cn/wiki/...，填写 wiki 路径后的 token，脚本会自动转换。
FEISHU_BITABLE_TABLE_IDtblXXXXXXXX
FEISHU_BITABLE_URL完整表格链接，用于群消息跳转。

3. 创建飞书自建应用

  1. 进入 飞书开放平台。
  2. 创建企业自建应用，应用名称可填 每日论文推荐助手。
  3. 进入应用后台，复制 App ID 和 App Secret。
  4. 进入权限管理，搜索并开通多维表格相关权限。
  5. 保存权限后发布应用版本；如果企业要求管理员审批，需要等审批完成。

权限建议

飞书权限名称可能随控制台版本调整。原则是：应用需要能查看多维表格、查询记录、新增记录、更新记录。可以在权限管理中搜索这些关键词：

  • 多维表格
  • bitable
  • base
  • record
  • 记录

如果多维表格链接是 feishu.cn/wiki/...，还需要开通任一 wiki 节点读取权限，例如：

  • wiki:node:read
  • wiki:wiki:readonly
  • wiki:wiki
如果后续运行时报 Forbidden、permission denied 或飞书错误码类似 91403，通常是权限或“添加文档应用”未完成。

把应用加入多维表格

  1. 回到刚创建的多维表格。
  2. 点击右上角更多菜单。
  3. 找到 添加文档应用 或类似入口。
  4. 选择 每日论文推荐助手。
  5. 授予可编辑权限。
这是最容易漏的一步。只在开放平台开 API 权限不够，目标多维表格也必须显式添加该应用。

4. 配置飞书群机器人

  1. 打开目标飞书群。
  2. 进入群设置，找到群机器人。
  3. 添加机器人，选择自定义机器人。
  4. 机器人名称建议填 每日论文推荐。
  5. 复制 webhook 地址，填入 FEISHU_WEBHOOK_URL。
  6. 建议开启签名校验，复制签名密钥，填入 FEISHU_SIGN_SECRET。

安全设置建议

  • 推荐开启签名校验。
  • 如果启用关键词校验，关键词填 每日论文推荐，脚本推送标题会包含这个词。
  • 除非运行机器有固定出口 IP，否则不要先开启 IP 白名单。

5. 配置本地环境

安装依赖

cd /mnt/d/faceid/code/paper-share
python3 -m pip install -r requirements.txt

确认本地 Codex 可用

command -v codex
codex --help

如果 codex 未登录或无法调用模型，先执行：

codex login
codex doctor

填写 .env

cd /mnt/d/faceid/code/paper-share
cp .env.example .env

将 .env 填成下面格式：

FEISHU_APP_ID=cli_xxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxx
FEISHU_BITABLE_APP_TOKEN=
FEISHU_BITABLE_WIKI_TOKEN=wiki_link_token
FEISHU_BITABLE_TABLE_ID=tblxxxxxxxxxxxxx
FEISHU_BITABLE_URL=https://xxx.feishu.cn/base/appxxxxxxxxxxxxx?table=tblxxxxxxxxxxxxx
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxx
FEISHU_SIGN_SECRET=xxxxxxxxxxxxx
SEMANTIC_SCHOLAR_API_KEY=
SEMANTIC_SCHOLAR_API_KEY 可先留空。留空时仍可跑 arXiv 抓取，只是补充 citation、venue、TLDR 等信息可能不稳定。

按需调整研究方向

编辑：

tools/paper_recommender/config.yaml

常调字段：

  • ranking.keyword_tiers.A：A 档强相关关键词，优先推荐。
  • ranking.keyword_tiers.B：B 档高相关关键词，联合命中时优先。
  • ranking.keyword_tiers.C：C 档专项关注关键词，用于扩大召回。
  • ranking.tier_weights：标题、摘要、元信息的分层权重。
  • ranking.negative_keyword_weights：不相关方向降权词和权重。
  • ranking.final_limit：每日推荐篇数。
  • arxiv.days_back：候选时间窗口。

6. 分阶段验证

按顺序执行，前一步成功后再进入下一步。

步骤 1：离线样例，不调用 Codex，不写飞书

python3 tools/paper_recommender/run_daily.py --dry-run --no-codex --sample

期望结果：终端输出 JSON，包含一篇样例论文推荐。

步骤 2：离线样例，调用本地 Codex，不写飞书

python3 tools/paper_recommender/run_daily.py --dry-run --sample

期望结果：输出中文摘要，日志中出现 Codex summaries generated。

步骤 3：离线样例，写多维表格，不推群

写入前可以先做只读权限检查：

python3 tools/paper_recommender/run_daily.py --check-feishu
python3 tools/paper_recommender/run_daily.py --sample --no-send

期望结果：飞书多维表格新增或更新一条样例论文记录。

步骤 4：离线样例，写表格并推群

python3 tools/paper_recommender/run_daily.py --sample

期望结果：多维表格记录的 是否已推送 更新为 是，飞书群收到卡片。

步骤 5：真实论文 dry-run

python3 tools/paper_recommender/run_daily.py --dry-run --no-codex

期望结果：能从 arXiv 获取真实候选。如果出现 HTTP 429，说明 arXiv 临时限流，稍后重试即可。

步骤 6：真实论文写表格但不推群

python3 tools/paper_recommender/run_daily.py --no-send

步骤 7：真实论文完整推送

python3 tools/paper_recommender/run_daily.py

7. 配置每日定时任务

确认手动完整推送成功后再加 cron。

crontab -e

添加北京时间每天 09:30 执行：

30 9 * * * cd /mnt/d/faceid/code/paper-share && python3 tools/paper_recommender/run_daily.py >> logs/paper_recommender.log 2>&1

查看日志：

tail -n 120 /mnt/d/faceid/code/paper-share/logs/paper_recommender.log
cron 环境变量通常比交互 shell 少。如果 cron 找不到 python3 或 codex，把命令改成绝对路径，例如 /home/wjw/miniconda3/bin/python3 和配置中的 codex.command 绝对路径。

8. 常见问题排查

现象优先检查处理方式
飞书多维表格写入失败，提示无权限 应用权限、版本发布、添加文档应用 确认开放平台权限已审批，并在目标多维表格中添加应用且授予编辑权限。
字段不存在或字段类型错误 字段名和字段类型 按本文字段表逐项核对。字段名不要多空格，不要把文本字段建成日期或复选框。
飞书群收不到消息 webhook、签名密钥、关键词校验 确认 FEISHU_WEBHOOK_URL 正确。若启用签名，确认 FEISHU_SIGN_SECRET 正确。若启用关键词，确认关键词为 每日论文推荐。
Codex 摘要失败但流程继续 Codex 登录、网络、模型配置 执行 codex doctor。脚本会自动降级为规则摘要，这是预期容错。
arXiv 报 HTTP 429 arXiv 限流 稍后重试。脚本已配置重试退避，也可降低 arxiv.max_results。
cron 手动能跑，定时不能跑 工作目录、绝对路径、环境变量 在 cron 中使用完整 cd，并把 python3、codex 换成绝对路径。

9. 最终检查清单

  • 飞书多维表格字段已按本文创建，字段名完全一致。
  • 飞书自建应用已创建，并拿到 APP_ID 和 APP_SECRET。
  • 多维表格查询、新增、更新相关权限已开通并发布。
  • 目标多维表格已添加该文档应用，并授予编辑权限。
  • 飞书群自定义机器人已创建，webhook 和签名密钥已复制。
  • .env 已填写所有飞书参数。
  • --dry-run --no-codex --sample 已通过。
  • --dry-run --sample 已通过，Codex 能生成中文摘要。
  • --sample --no-send 已成功写入多维表格。
  • --sample 已成功推送飞书群。
  • 去掉 --sample 后，真实论文流程已验证。
  • cron 已配置，日志路径可正常写入。

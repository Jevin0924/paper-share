一句话总结

HarmVideoBench 将有害视频评测从二分类检测推进到三层诊断推理：先看模型能否识别显性证据，再看能否理解片段内部含义，最后看能否结合文化、历史和社会语境进行边界受控的外部推理；在 1,379 个视频、4,137 道多选题上，当前模型的主要短板集中在 Beyond-Clip Reasoning，而作者提出的 BCR 将 Qwen3-VL-8B-Instruct 基线的宏平均从 61.7% 提升到 84.4%（见 PAGE 1、PAGE 3、PAGE 7）。

1. 论文问题与贡献定位

本文研究的是大型视觉语言模型（LVLMs）在有害视频理解中的评测问题。与常规视频问答或动作识别不同，有害视频审核并不只要求模型“看见了什么”，还要求模型解释“为什么这段内容会造成伤害”。论文明确指出，现有有害视频基准存在两个核心不足：一是忽略有害视频的多层特征，常把任务简化为 harmful / not harmful 的二分类；二是缺少解释性理由，只评估模型是否标记正确，而不评估其判断依据（见 PAGE 1、PAGE 3）。

这种问题设定非常关键。真实视频中的伤害往往不是单一视觉标签能够覆盖的：有的伤害来自画面中直接可见的暴力、辱骂或危险行为；有的伤害来自剪辑、字幕、旁白共同构成的羞辱、煽动或污名化叙事；还有的伤害依赖特定社区、历史语境或平台文化。如果评测只问“这个视频是否有害”，模型可能依赖表面词、显性图像或数据集捷径取得高分，却并未真正理解有害性。

论文提出 HarmVideoBench 的核心贡献，是把有害视频理解拆成三个递进层级：Observable Evidence、Clip-Internal Meaning 和 Beyond-Clip Reasoning。每个视频配三道题，分别对应这三个层级，因此 1,379 个视频形成 4,137 道多选题（见 PAGE 3、PAGE 5、PAGE 21）。这一设计让错误诊断从“模型错了”变成“模型在哪个理解层面错了”。

此外，论文还提出 BCR（Boundary-Constrained Reasoning），一种与基准结构对齐的轻量方法。BCR 的思想不是简单延长推理链，而是先预测问题的证据边界：有些问题只应依据视频片段内部证据作答，有些问题才需要引入外部语境。只有当问题需要 Beyond-Clip Reasoning 时，BCR 才从训练侧记忆库中检索上下文片段（见 PAGE 6、PAGE 8、PAGE 25）。

2. 背景与动机

视频平台已经成为娱乐、教育和社会交互的主要载体，论文以 YouTube、Bilibili 等平台为背景，指出有害视频可能涉及暴力、错误信息、羞辱、针对文化或身份群体的攻击等风险（见 PAGE 1）。这类内容的治理需求推动了 LVLMs 在自动内容审核中的应用，因为 LVLMs 能同时处理视觉、文本、语音转写和开放式问答。

然而，论文强调，已有工作虽然覆盖了多模态安全评测，却往往把视频有害性理解压缩为粗粒度分类。HateMM、MultiHateClip 等视频基准关注真实有害视频，但主要以标签分类形式存在；MM-SafetyBench、MMSafeAware 等安全基准覆盖多种安全问题，却不一定直接基于真实有害视频，也缺少分层推理评估（见 PAGE 2 的 Table 1）。

这使得已有评测难以区分三类能力：第一，模型是否能识别视频中直接出现的辱骂、暴力、危险行为或符号；第二，模型是否能把多个视频内部线索整合成有害意图或有害框架；第三，模型是否知道某些短语、姿态、梗图、历史符号或社会后果需要外部背景才能判断。论文认为，第三类能力正是当前模型的深层短板（见 PAGE 3、PAGE 8、PAGE 9）。

更重要的是，缺少解释性 rationale 会把评测变成黑箱。论文指出，模型可能通过特定负面词、显性标签或视觉捷径获得高准确率；一旦出错，研究者无法判断错误源于视觉感知、语义整合，还是外部语境缺失（见 PAGE 3）。HarmVideoBench 的三层结构正是为了解决这个诊断问题。

因此，HarmVideoBench 不是一个单纯更大的视频安全数据集，而是一个以“推理层级”为中心的诊断基准。它的目标不是替代内容审核政策，而是测量模型能否从可见证据走向片段含义，再走向有边界的上下文推理。

4. 预备知识：三层推理 taxonomy

Observable Evidence 指模型能否识别视频或转写中直接可见、可听、可读的有害证据，例如画面中的攻击行为、屏幕上的仇恨标语、语音中的侮辱性表达或具有明确指向的符号。论文强调，这一层不要求外部背景知识，因为答案应当直接来自视频中的显性证据（见 PAGE 5）。

Clip-Internal Meaning 指模型能否把片段内部的多个模态线索整合成有害含义。它不只是“看见一个词”或“识别一个动作”，而是理解视频通过旁白、剪辑、字幕、视觉并置等方式表达的羞辱、嘲讽、暴力美化或煽动意图。答案仍然应来自片段和转写本身，不需要额外社会背景（见 PAGE 5、PAGE 6）。

Beyond-Clip Reasoning 指模型需要超出视频内部证据，连接文化、历史、社会或后果相关背景才能正确判断。例如某些手势、梗、短语或讽刺框架可能只有在特定社区或历史语境中才具有有害含义；某些看似普通的行为在社会传播语境中可能鼓励危险模仿。论文认为这一层最能暴露当前 LVLMs 的深层推理短板（见 PAGE 6）。

附录进一步把边界规则操作化：如果答案直接来自“看见或听见了什么”，属于 Observable Evidence；如果答案能由视频和转写内部推断，属于 Clip-Internal Meaning；如果答案需要额外文化、历史、社会或社区知识，属于 Beyond-Clip Reasoning（见 PAGE 22、PAGE 23 的 Table 4）。这使 taxonomy 不是主观印象，而是可执行的标注规则。

5. 数据构建：从公开视频到三层问答

HarmVideoBench 的数据来源包括已有有害视频资源和平台公开视频。论文称，其从 HateMM、MultiHateClip 等资源出发，并补充 YouTube、Bilibili 以及公开审核档案中的视频，形成约 1,900 个初始候选视频；最终保留 1,379 个视频（见 PAGE 4、PAGE 19、PAGE 21）。这些视频必须具有公开可审计来源、足够上下文以及可由人工验证的语言覆盖。

主题覆盖包括 violence / abuse、hate / identity-targeted harassment、misinformation / manipulation、sexual exploitation / coercion、dangerous acts / self-harm encouragement、bullying / humiliation。PAGE 21 的 Table 3 给出具体分布：暴力/虐待 330 个视频，占 23.9%；仇恨/身份定向骚扰 287 个，占 20.8%；错误信息/操纵 222 个，占 16.1%；性剥削/胁迫 211 个，占 15.3%；危险行为/自伤鼓励 192 个，占 13.9%；霸凌/羞辱 137 个，占 9.9%。

语言分布也很重要。Table 3 显示，英语视频 770 个，占 55.8%；中文视频 393 个，占 28.5%；混合或其他语言视频 216 个，占 15.7%。平均时长为 92.0 秒（见 PAGE 21）。这说明 HarmVideoBench 并非纯英语安全基准，但作者也在 Limitations 中承认，英语和中文占主导，低资源语言覆盖不足（见 PAGE 12）。

标注流程采用 AI 辅助候选生成和人工复核。Qwen-2.5-VL-72B 先生成详细 caption 和三类问题候选，但论文明确强调这些输出只是 candidates，不是 gold annotations。随后四名训练过的标注者和一名 senior annotator 进行人工复核、编辑、重写、重分配类别或丢弃（见 PAGE 4、PAGE 5、PAGE 19）。

从 5,344 个候选题项中，最终保留 4,137 个，多选题保留率为 77.41%。论文还报告，12.6% 的机器候选需要 senior adjudication 处理 unresolved level 或 grounding disagreement；仅 37.7% 的保留项只需 minor edits，而 44.0% 需要对问题、答案或干扰项进行实质性人工重写（见 PAGE 5）。这组数字支持作者关于“最终基准是 human-curated，而非原始 LMM 输出集合”的主张。

6. 形式化表达与公式证据

证据说明：给定全文中没有出现完整的模型训练目标函数或正式数学公式推导。因此，以下公式均为论文中明确数值、算法记号或实验设置的形式化转写，并非作者原文提出的新增理论公式；无法确认的公式部分记为证据不足。

第一，数据规模可由论文的构造规则直接写成：

$$N_{\text{MCQ}} = N_{\text{video}} \times 3 = 1379 \times 3 = 4137$$

这表示每个视频分别生成 Observable Evidence、Clip-Internal Meaning、Beyond-Clip Reasoning 三道题，因此形成 4,137 个多选题项。证据见 PAGE 1、PAGE 3、PAGE 21。

第二，论文给出视频级划分为 70/10/20，并报告具体数量：

$$N_{\text{train}}/N_{\text{dev}}/N_{\text{test}} = 965/138/276$$

这说明训练、开发、测试划分在视频级完成，避免同一视频的问题泄漏到不同 split。证据见 PAGE 7。

第三，BCR 相对 Qwen3-VL-8B-Instruct 基线的宏平均提升可以写为：

$$\Delta_{\text{Macro}} = 84.4 - 61.7 = 22.7\ \text{points}$$

这不是一个小幅 prompt trick，而是从弱基线到 task-adapted 结构化方法的显著提升。证据见 PAGE 1、PAGE 7。

第四，在 Beyond-Clip Reasoning 上，BCR 相对 HarmVideo-SFT 的直接增益为：

$$\Delta_{\text{BCR,SFT}}^{\text{Beyond}} = 70.1 - 64.8 = 5.3\ \text{points}$$

这说明 BCR 的主要价值并不只是微调带来的任务适配，而是 scope prediction、selective retrieval 与 boundary-constrained decoding 对深层推理的额外贡献。证据见 PAGE 7、PAGE 29。

第五，训练细节中给出了 LoRA 训练学习率与损失权重：

$$\eta_{\text{peak}} = 2 \times 10^{-4}, \quad \lambda_1 = 0.2,\quad \lambda_2 = 0.1$$

这里 $\eta_{\text{peak}}$ 表示峰值学习率，$\lambda_1$ 和 $\lambda_2$ 是 BCR 训练中在开发集上选择的损失权重。证据见 PAGE 25。

第六，BCR 的检索设置可形式化为：

$$|M_{\text{train}}| = 3316,\quad k = 3$$

其中 $M_{\text{train}}$ 是由训练集解释、上下文注释和影响描述构成的训练侧记忆库，$k=3$ 表示仅对 Beyond-Clip Reasoning 问题检索 top-3 支持片段。证据见 PAGE 25。

7. 方法详解：HarmVideoBench 的评测框架

7.1 问题一：二分类为什么不够

传统 harmfulness evaluation 通常以单一问题开始：“Is this video harmful?” Figure 1 对此进行了图示比较：传统评测从原始视频得到二元决策，而 HarmVideoBench 将同一原始视频转化为三层 reasoning evaluation（见 PAGE 2）。这种差异决定了评测目标不同：前者测分类，后者测理解结构。

二分类的问题在于，它把错误源混在一起。例如模型可能能看见辱骂词，却无法理解视频剪辑如何强化羞辱；也可能能解释片段内部意义，却不知道某个符号在特定历史语境中为何具有攻击性。二分类只给出最终对错，不暴露这些分层错误。

7.2 创新一：Observable Evidence 作为基础感知层

Observable Evidence 的问题是：模型是否能准确识别视频中直接出现的有害线索。论文列举的证据类型包括视觉场景、口头话语、屏幕文字和符号线索。若视频中有可见攻击、屏幕上的仇恨标语或针对群体的口头侮辱，模型应当直接从多模态证据中识别出来（见 PAGE 5）。

这个层级的意义在于建立诊断基线。若模型在 Observable Evidence 上失败，那么后续 Clip-Internal Meaning 或 Beyond-Clip Reasoning 的错误可能只是感知失败的连锁结果。Table 2 显示，强模型在这一层接近人类，但仍未完全饱和：Human 为 98.9，Gemini-3.1-Pro 为 96.0，BCR 为 96.7（见 PAGE 7）。

7.3 创新二：Clip-Internal Meaning 评估片段内部语义整合

Clip-Internal Meaning 不再问“有没有某个显性线索”，而是问视频整体表达了什么有害含义。论文给出的例子包括：视频没有直接说出 slur，但可能通过嘲讽旁白、剪辑选择或并置字幕传达羞辱、嘲弄、暴力美化或煽动（见 PAGE 5、PAGE 6）。

这一层要求模型连接多个模态证据。它与 Observable Evidence 的差异在于，答案不一定是单个显性对象，而是由画面、字幕、语音和剪辑共同构成的 proposition。论文在结果分析中指出，最强 off-the-shelf 模型在 Clip-Internal Meaning 上仍比人类低 12.9 分，说明即使感知较可靠，视频内部语义理解仍是瓶颈（见 PAGE 8）。

7.4 创新三：Beyond-Clip Reasoning 评估上下文推理

Beyond-Clip Reasoning 是 HarmVideoBench 最具诊断价值的部分。论文明确说，这一层的问题不能仅凭可见和可听内容解决，模型必须把视频连接到文化、历史、社会或后果相关语境（见 PAGE 6）。这与很多现有视频基准不同，因为它要求模型知道“什么时候视频内部证据不够”。

结果也显示，Beyond-Clip Reasoning 是当前模型的主要短板。Table 2 中，Human 为 96.2，而最强 off-the-shelf 模型 Gemini-3.1-Pro 为 65.8，差距达 30.4 分；Qwen3-VL-8B-Instruct 仅 48.1；BCR 提升到 70.1，但仍距人类 26.1 分（见 PAGE 7、PAGE 8）。这说明外部语境推理不是通过模型规模自然解决的问题。

7.5 方法创新四：BCR 的证据边界控制

BCR 的核心假设是，许多有害视频错误来自模型不尊重问题的 evidence boundary。有些问题只需片段内部证据即可回答，而有些问题需要外部语境。若模型把所有问题都当成“开放式道德评论”，就容易生成泛泛而谈的危害；若模型把所有问题都限制在画面内部，又无法回答文化语境问题（见 PAGE 6）。

Algorithm 1 描述了 BCR 推理流程：输入视频帧 $F$、转写 $T$ 和问题 $q$；先预测目标问题类型以及是否 clip-grounded 或 beyond-clip；选择格式相关的帧和转写证据 $(F^\ell,T^\ell)$；若问题需要 Beyond-Clip Reasoning，则从训练侧 memory bank 检索 top-k 上下文片段 $R^\ell$，否则令 $R^\ell=\emptyset$；最后在边界约束的中间链上解码答案（见 PAGE 8）。

这一路径的关键不是“检索越多越好”，而是“只在需要时检索”。PAGE 25 进一步说明，BCR 的记忆库由训练集解释、curator-written context notes 和简洁 impact descriptions 构成，共 3,316 个训练侧条目，并且只对 Beyond-Clip Reasoning 问题检索 top-3 片段。

7.6 BCR 的三个组件

论文将 BCR 分为 reasoning-scope prediction、selective context augmentation 和 boundary-constrained decoding 三部分。第一部分判断问题是否应停留在 clip-grounded 范围，还是进入 beyond-clip；第二部分只在必要时引入检索上下文；第三部分要求模型先预测片段内部输出，再在此基础上作 beyond-clip 判断（见 PAGE 6、PAGE 25）。

这三个组件分别针对三种典型错误：Context Omission、Unsupported Harm Leap 和 Generic Moralization。Context Omission 是模型没有引入必要背景；Unsupported Harm Leap 是模型从可见线索跳到没有证据支持的下游伤害；Generic Moralization 是模型输出泛泛道德评论而未回答具体问题。Table 5 显示，BCR 将 Context Omission 从 31.5% 降到 22.0%，Unsupported Harm Leap 从 28.0% 降到 19.5%（见 PAGE 24）。

8. 实验设置

实验包含两个 reference baselines：Random 与 Human。Random 为均匀随机选择选项，作为下界；Human 为四名训练标注者在相同评测集上的平均表现，作者说明这是 informed reference，而不是普通人群估计（见 PAGE 6）。Human 在三层上分别达到 98.9、97.5、96.2，宏平均 97.5（见 PAGE 7）。

模型覆盖 5 个闭源系统、14 个开源基线，以及两个 task-adapted baselines。闭源系统包括 Gemini-3.1-Pro、Gemini-3.1-Flash、GPT-5.5、Claude-Opus-4.7 和 Claude-Sonnet-4.6；开源模型包括 Qwen3-VL 系列 thinking / instruct 模型、Phi-4-reasoning-vision、MiniCPM-V、Lance、LFM、GLM、Kimi 等（见 PAGE 7、PAGE 28）。

推理设置方面，论文设 temperature 为 0.0，并对所有模型使用 greedy decoding。每个视频由 12 个采样帧和 transcript evidence 表示；原生多模态模型直接消费采样帧，text-first 或 hybrid 模型接收标准化的 frame captions、OCR snippets 和 transcripts 序列化表示（见 PAGE 7）。

Task-adapted baseline HarmVideo-SFT 基于 Qwen3-VL-8B-Instruct，输入采样帧、转写证据和短类别描述，一次预测一个目标问题。训练细节在 PAGE 25 给出：LoRA rank 16，训练三轮，峰值学习率 $2\times10^{-4}$，有效 batch size 64，并以开发集宏平均 early stopping。

第一条结论是明显的层级退化。无论闭源、开源、thinking、non-thinking 还是 task-adapted 模型，性能都大致从 Observable Evidence 到 Clip-Internal Meaning 再到 Beyond-Clip Reasoning 递减。论文把这称为 “Hierarchical failure is the central benchmark signal”（见 PAGE 9）。这说明 HarmVideoBench 的难点不是普通检测，而是推理深度。

第二条结论是，强模型的感知能力已经较强，但远未达到深层理解。Gemini-3.1-Pro 在 Observable Evidence 上达到 96.0，仅比 Human 低 2.8 分；但在 Beyond-Clip Reasoning 上为 65.8，比 Human 低 30.4 分（见 PAGE 7、PAGE 8）。这表明模型并非完全看不懂视频，而是无法稳定把视频证据连接到外部社会语境。

第三条结论是，BCR 在 compact 8B backbone 上取得了最强宏平均。Qwen3-VL-8B-Instruct 的 Macro Avg. 为 61.7，HarmVideo-SFT 为 80.5，BCR 为 84.4（见 PAGE 7）。这说明任务适配已经能显著缩小差距，而进一步引入 evidence boundary 和 selective retrieval 能带来额外增益。

但第四条结论同样重要：BCR 没有解决深层审核问题。它在 Beyond-Clip Reasoning 上达到 70.1，但 Human 是 96.2，仍有 26.1 分差距（见 PAGE 7）。论文也在 Finding 2 中指出，benchmark-aligned structure helps, but only locally；轻量适配能缩小差距，但不能关闭差距（见 PAGE 9）。

消融实验表明，BCR 的收益主要来自深层推理稳定性，而不是低层感知微调。去掉 routing 后 Beyond-Clip 从 70.1 降到 66.0；去掉 retrieval 后降到 66.8；去掉 boundary constraints 后降到 66.5（见 PAGE 9、PAGE 29）。这些下降幅度均集中在 Beyond-Clip Reasoning，符合方法设计预期。

这组结果也说明，单纯把任务格式教给模型并不足够。single-stage prompt 或 HarmVideo-SFT 已经能显著提升 8B 基线，但 BCR 进一步要求模型在回答前明确证据边界。对内容安全场景而言，这比一般 CoT 更重要，因为错误的外推可能导致过度审核或错误归因。

11. CoT、规模与语言差距

论文对 chain-of-thought prompting 的分析显示，CoT 对非 thinking 模型有小幅帮助，但对 thinking-mode checkpoint 几乎没有收益。Table 9 中，Qwen3-VL-8B-Instruct 从 59.6 到 61.7，提升 2.1；Qwen3-VL-30B-A3B-Instruct 从 74.0 到 76.3，提升 2.3；但 Qwen3-VL-235B-A22B-Thinking 从 80.2 到 80.1，略降 0.1（见 PAGE 10、PAGE 29）。这说明外部 CoT 不是解决 Beyond-Clip Reasoning 的充分方案。

规模分析也有类似结论。Qwen3-VL-8B-Instruct 宏平均为 61.7，Qwen3-VL-32B-Instruct 为 79.0，Qwen3-VL-235B-A22B-Instruct 为 79.7；规模提升显著改善整体表现，但 Beyond-Clip Reasoning 从 48.1 提升到 62.0 后仍远低于 Human 的 96.2（见 PAGE 29）。因此，模型规模可以改善基础能力，却不能自然补足社会语境推理。

语言差距是另一个关键诊断。论文指出 English-only evaluation 一般提高各类别分数，而 Chinese-only evaluation 在 Beyond-Clip Reasoning 上下降最明显（见 PAGE 9、PAGE 11、PAGE 12）。作者解释，这可能来自中文视频中更多文化特定 slang、平台惯例和历史语境，当前 LVLMs 难以仅凭视频证据恢复这些背景（见 PAGE 23、PAGE 24）。

12. 错误类型：模型为什么失败

论文把 HarmVideo-SFT 和 BCR 的 200 个采样错误分为五类：Evidence Miss、Meaning Collapse、Context Omission、Unsupported Harm Leap 和 Generic Moralization / Other（见 PAGE 23、PAGE 24）。这比单纯报告 accuracy 更有价值，因为它把“错误答案”拆成可解释的认知失败。

Context Omission 表示模型没有引入必要背景，尤其影响 Beyond-Clip Reasoning。Unsupported Harm Leap 则相反，模型从一个可见线索跳到过宽、过重或无证据支持的下游危害。Meaning Collapse 表示模型把更深层含义简化为片段内部表述或直接 paraphrase。Generic Moralization 则是输出抽象道德评论，但没有回答具体 harmful mechanism。

Table 5 显示，BCR 有效减少了 Context Omission 和 Unsupported Harm Leap，但 Generic Moralization / Other 比例上升到 33.0（见 PAGE 24）。这并不必然表示 BCR 更差，而可能是因为前两类结构性错误减少后，剩余错误更多集中在残余泛化或其他难以分类的推理问题。论文对此没有给出更细二级分类，因此更具体原因证据不足。

Figure 7 提供若干代表性错误。一个 Gemini-3.1-Pro 例子中，模型对含反黑人种族主义歌词的案例给出条件式泛泛模板，没有识别实际 anti-Black racist lyrics 和 dehumanizing mechanism；另一个模型错误识别目标群体，把实际 anti-Black slurs 误解为 Native American stereotype 相关问题；还有例子把涉及亲密关系边界侵犯的问题误解为医疗不公或游戏场景（见 PAGE 24）。这些案例说明，错误往往不是随机选项错误，而是目标识别、语义框架和证据边界失败。

13. 代码与可复现性状态

论文首页标注 “Code Repository: Coming soon”，但给定元信息中 code_url 与 project_url 为空；全文也未提供可访问的 GitHub URL（见 PAGE 1）。因此，本文未提供可确认的公开代码，不能进行源码文件、函数路径或实现片段对应分析。

可复现性方面，论文提供了相当多的实验设置信息：temperature 0.0、greedy decoding、每视频 12 帧、视频级 70/10/20 划分、HarmVideo-SFT 的 LoRA rank、训练轮数、学习率、batch size、早停指标、BCR 记忆库大小与 top-3 检索策略（见 PAGE 7、PAGE 25）。这些信息足以理解方法轮廓，但在代码未发布、数据获取依赖公开视频与审核材料的情况下，完整复现实验仍证据不足。

尤其需要注意，BCR 的 retrieval bank 来自 training-side benchmark resources，而不是开放域证据库。作者在 Limitations 中明确指出，这意味着 BCR 的表现不能说明真实世界深度审核已经解决，因为真实部署需要模型自主检索并验证缺失的文化或上下文知识（见 PAGE 13）。

14. 局限分析

第一，语言与文化覆盖有限。作者自述 HarmVideoBench 中大多数视频为英语或中文，因此主要评估这些文化语境中的 harmfulness，其他语言代表性不足。未来版本应扩展语言覆盖，特别是低资源语言（见 PAGE 12）。这不仅是数据分布问题，也会影响 Beyond-Clip Reasoning，因为外部语境与本地文化强相关。

第二，基准假设输入来自预筛选 moderation queue，不评估 benign content 上的 false positives。作者明确指出，模型若倾向 abstain 或 over-refuse，可能在该设置中呈现偏斜表现；未来应加入具有挑战性的非有害视频，例如关于仇恨言论的纪录片，以联合评估深度理解和过度审核风险（见 PAGE 12）。

第三，harm labels 和 reference explanations 不可避免包含规范性选择。论文承认，即使有显式标注规则和 adjudication，也无法消除文化或社会价值假设；HarmVideoBench 应被视为 moderation-oriented diagnostic resource，而非 universal moral ontology（见 PAGE 12、PAGE 13）。这是内容安全基准必须正视的问题：评测的“正确答案”并不只是感知事实，也涉及社会解释边界。

第四，数据污染风险存在。由于数据来自 YouTube、Bilibili 等公开视频，一些视频可能已出现在闭源 LMM 的训练数据中。作者认为该风险是行业范围挑战，并指出 Beyond-Clip 类别要求评估细粒度干扰项和有界后果，不应被简单内容记忆完全解决（见 PAGE 12、PAGE 13）。不过，在闭源模型训练数据不可审计的情况下，污染影响大小仍证据不足。

第五，主评测依赖 MCQ。作者承认，多选题可能受 distractor elimination 和 positional bias 影响；MCQ 便于规模化评分，但不能替代生成式 justification。论文在附录提供 open-ended diagnostic protocol，但因自由形式评估依赖 judge models 且容易受措辞、verbosity 和 rhetoric 影响，作者将其限制在附录级分析（见 PAGE 13、PAGE 26）。

15. 对研究与业务的意义

对视频理解研究而言，HarmVideoBench 的价值在于把视频问答中的“看懂视频”细化为安全语境下的证据层、语义层和外部语境层。它提示研究者：在真实视频安全场景中，模型的失败不一定是视觉编码不够强，也可能是证据边界控制、文化知识检索和社会后果推理不足。

对内容审核系统而言，该论文有助于设计更可诊断的评估体系。实际系统不能只输出“违规/不违规”，还需要说明依据来自画面、转写、字幕、外部语境还是政策推理。HarmVideoBench 的三层 taxonomy 可以转化为审核模型的分层验收指标：先验感知，再验片段内部语义，最后验有界外部推理。

对检测、跟踪、ReID 等视觉任务团队而言，本文的直接技术相关性有限。论文没有提出新的检测器、轨迹建模或细粒度时空定位方法，也没有给出 frame-level box / temporal localization 标签。其核心价值更接近视频 VLM 评测、内容安全、多模态推理和审核解释体系。

16. 结论

HarmVideoBench 的核心贡献不是提出一个更复杂的排行榜，而是重新定义有害视频理解的评测单位。它将问题从“模型是否判定有害”转变为“模型是否知道证据在哪里、含义如何由片段构成、哪些判断需要外部语境”。Table 2、Table 7、Table 9 和 Table 10 共同说明，当前模型的主要瓶颈不在显性 cue recognition，而在 clip-internal semantics 与 beyond-clip contextual grounding（见 PAGE 7、PAGE 9、PAGE 29）。

BCR 证明了 benchmark-aligned structure 的价值：scope prediction、selective retrieval 与 boundary-constrained decoding 能显著提升 compact model 在深层问题上的表现。但它也暴露了剩余挑战：即使达到 84.4 宏平均，Beyond-Clip Reasoning 仍距人类很远；且 BCR 的检索依赖训练侧资源，尚不能代表开放世界内容审核能力（见 PAGE 7、PAGE 13、PAGE 25）。

有害视频理解的难点，不是让模型更快说“有害”，而是让模型知道自己凭什么、在什么边界内、基于哪些语境作出判断。

一句话总结：CUPID 是一种面向目标人物（Person-of-Interest, POI）的视频 Deepfake 检测方法，它把 3D 人脸重建得到的 UV 纹理图（UV texture map）输入 Masked Autoencoder（MAE），用真实视频学习身份一致的潜表示，并在测试时将待测视频与该 POI 的干净参考视频进行相似度匹配，同时输出 UV 空间残差图作为可解释证据（见 PAGE 1-6）。

从任务定位看，CUPID 属于“有目标人物参考视频”的身份一致性检测，而不是开放集通用 Deepfake 分类器。它的业务价值主要体现在名人、政要、企业高管等高风险人物的视频真伪审核：系统不要求为每个 POI 重新训练模型，而是在推理阶段使用该 POI 的少量真实参考视频建立身份分布（见 PAGE 1-2、PAGE 5）。这一设定与普通用户开放集审核不同，前提是业务侧能够取得足够干净且覆盖多姿态、多光照的参考视频。

本文最核心的技术判断是：UV 纹理图提供了跨身份、跨姿态的稠密语义对齐，而 MAE 的上下文重建目标能够从真实人脸纹理中学习身份相关的潜表示。两者结合后，CUPID 在压缩、缩放等后处理下比多数通用检测器更稳健，并且推理速度显著快于公开 POI 基线 poi-forensics（见 PAGE 2、PAGE 8-11）。

## 背景与动机

近年来，生成式 AI 使合成视频的逼真度显著提高，针对特定高知名度人物的 Deepfake 已经被用于政治误导、名人冒用和企业诈骗等场景。论文将这类目标人物称为 Person-of-Interest（POI），并明确把问题定义为保护特定人物免受合成视频冒用（见 PAGE 1）。这与普通“给一段视频判断真伪”的通用检测问题不同，因为 POI 任务可以利用目标人物的大量真实公开视频作为先验。

现有 POI 检测方法大体分为两类。第一类在训练阶段使用该 POI 的数据，学习该人物的外观、表情、动作或音视频动态特征；缺点是每个目标人物都需要单独训练检测器，并且需要大量该人物数据（见 PAGE 1）。第二类在训练阶段学习通用身份表示，推理时再输入该 POI 的真实参考视频与待测视频，通过潜空间相似度进行判别；ID-Reveal 和 poi-forensics 属于这类思路（见 PAGE 1）。CUPID 选择第二类路线，因此更接近可扩展部署的 POI 检测系统。

论文认为，现有 POI 方法仍未同时解决三个要求：后处理鲁棒性（robustness to post-processing）、计算效率（efficiency）和可解释性（interpretability）（见 PAGE 1-2）。这三个要求在真实业务中并非附加项。在线视频通常经过压缩、缩放、转码；审核系统需要低延迟处理大量视频；风险处置还需要解释“哪些脸部区域导致了伪造判断”。如果检测器只能输出一个黑箱分数，后续人工复核和风控解释都会受限。

CUPID 的出发点是把检测空间从原始帧空间迁移到 UV 纹理空间。UV texture map 是通过 3D Morphable Model（3DMM）把人脸外观投影到规范化二维纹理坐标系后的图像表示；在这个空间中，同一像素位置对应相同语义人脸区域，例如眼睛、鼻尖或嘴部，而不再强烈受原始视频中姿态和表情变化影响（见 PAGE 2-3）。这为身份一致性建模和残差可视化提供了结构化基础。

论文进一步用 MAE 学习 UV 纹理图的潜表示。MAE 通过随机遮挡图像 patch 并重建缺失部分，迫使模型依赖全局上下文而不是局部像素复制。CUPID 将这一机制用于真实视频中的 UV 纹理图，使模型在不使用 Deepfake 样本、不包含目标 POI 训练数据的条件下学习身份相关特征（见 PAGE 2-4）。这也是论文声称“训练阶段不需要伪造视频、不需要目标 POI 入库”的主要依据（见 PAGE 1-2）。

## 预备知识

### 3DMM 与 UV 纹理图

3D Morphable Model（3DMM）假设不同人脸 3D 扫描之间存在稠密顶点对应：同一个顶点索引在不同人脸上代表同一个语义位置。例如第 $i$ 个顶点如果在一张脸上是鼻尖，那么在其他脸上也对应鼻尖区域（见 PAGE 2）。令 $S=\{S_1,S_2,\ldots,S_N\}$ 表示 $N$ 个示例 3D 人脸，每个 $S_i\in\mathbb{R}^{3\times M}$ 包含 $M$ 个顶点的三维坐标，则新的人脸形状可以表示为示例脸的线性组合：

$$
S(\alpha)=S_1\alpha_1+S_2\alpha_2+\cdots+S_N\alpha_N
$$

其中 $\alpha=[\alpha_1,\alpha_2,\ldots,\alpha_N]^T$ 是形状组合系数。这个公式说明，3DMM 用真实人脸样本张成一个可行的人脸形状空间；在本文中，它为从二维图像恢复规范化人脸结构提供几何先验（见 PAGE 2，公式 1）。

给定 3D 形状 $S(\alpha)$，论文用正交投影矩阵 $T\in\mathbb{R}^{2\times3}$、旋转矩阵 $R\in\mathbb{R}^{3\times3}$、尺度因子 $s$ 和二维平移 $t_{2D}\in\mathbb{R}^2$ 将其投影到图像平面：

$$
S_{2D}=sTRS(\alpha)+t_{2D}
$$

其中 $S_{2D}\in\mathbb{R}^{2\times M}$ 是二维投影顶点。直观地说，这个公式描述了从规范 3D 人脸到当前图像中人脸位置的几何变换；估计这些参数后，就可以把图像中的颜色采样回统一的 UV 坐标系（见 PAGE 2，公式 2）。

UV 纹理图 $I_{UV}$ 是把原始人脸图像 $I$ 中的人脸颜色映射到二维纹理坐标 $(u,v)$ 后得到的图像。论文给出的采样关系为：

$$
[I_{UV}]_{u,v}=[I]_{x,y}
$$

其中 $(x,y)$ 是原始图像中的二维人脸投影点，$(u,v)$ 是对应 UV 坐标。这个公式的含义是：UV 图上的每个像素记录了原始图像中相应人脸位置的颜色，因此不同帧、不同身份的人脸可以被放到同一个语义纹理平面上比较（见 PAGE 2-3，公式 3）。

### MAE 的基本机制

Masked Autoencoder（MAE）是一种自监督视觉表征学习框架。给定图像 $X\in\mathbb{R}^{H\times W\times C}$，其中 $H$、$W$、$C$ 分别表示高、宽和通道数，MAE 将图像划分为 $N$ 个不重叠 patch，并随机选择可见 patch 集合 $V$ 与被遮挡 patch 集合 $M$。遮挡率定义为 $\rho=|M|/N$（见 PAGE 3）。

MAE 编码器 $E(\cdot)$ 只处理可见 patch：

$$
z=E(x_V)
$$

其中 $x_V$ 表示可见 patch 序列，$z$ 是编码器输出的潜表示。这个公式说明，编码器不能直接看到完整图像，必须从局部可见信息推断全局结构（见 PAGE 3，公式 4）。

随后，轻量解码器 $D(\cdot)$ 把潜表示与 mask token 结合，预测完整 patch 序列：

$$
\hat{x}=D(z,M)
$$

其中 $\hat{x}$ 是重建结果，$M$ 指示缺失 patch 的位置。训练损失只在被遮挡 patch 上计算，以避免模型简单复制可见区域；这也是 MAE 适合学习全局语义和结构关系的原因（见 PAGE 3，公式 5）。

## 方法详解

### 1. 整体技术路线：从原始视频到 POI 身份相似度

CUPID 的输入是视频帧，首先通过人脸检测、对齐和 3D 重建得到 UV 纹理图，然后用 MAE 编码器提取身份相关 embedding。训练阶段只使用真实视频；测试阶段，对于某个 POI，系统从若干干净参考视频中采样帧，建立 reference embedding 集合，再将待测视频的 embedding 与 reference 集合比较，得到真伪分数（见 PAGE 3-5）。

这一设计与传统帧级 Deepfake 分类器的差异在于，CUPID 不直接学习“真实/伪造”的二分类边界，而是学习“真实人脸身份纹理在 UV 空间中的一致性”。在推理时，如果待测视频确实属于该 POI，其 embedding 应该接近该 POI 的真实参考分布；如果视频被换脸或身份生成过程破坏，embedding 会偏离参考分布（见 PAGE 5）。

论文明确指出，CUPID 的训练不需要 Deepfake 视频，也不需要把测试 POI 包含进训练集。训练数据来自真实视频中的 UV 纹理图，目标是学习可泛化到未见身份的潜空间（见 PAGE 1-2、PAGE 3-4）。这使它更适合缺少伪造样本但拥有真实人物素材的场景，例如新闻人物、企业高管或主播的冒用检测。

### 2. UV 纹理图作为输入：消除姿态与表情干扰

训练阶段，给定某个身份的视频，CUPID 均匀采样 $N_{frm}$ 帧，并从每帧中提取对齐人脸 crop。随后，3DMM-based UV extractor 拟合人脸模型，将人脸外观投影到规范 UV 坐标空间，为每帧生成一个 UV texture map $I_{UV}$（见 PAGE 3）。这里 $N_{frm}$ 表示每个视频采样的帧数。

论文强调，CUPID 与传统 MAE 图像建模的关键差异正在于输入不是原始帧，而是 UV texture map。原始人脸帧中，眼睛、鼻子、嘴巴的位置会随姿态、尺度和表情变化；UV 空间中，对应区域的位置稳定，因此更适合做身份一致性比较（见 PAGE 2-3）。Fig. 2 展示了从输入人脸图像到 3D 重建再到 UV texture map 的过程，Fig. 3 对比了普通人脸图像与 UV texture map 中眼睛、鼻子、嘴巴区域的语义对齐效果（见 PAGE 2-3）。

从检测角度看，这种输入表示有两个直接收益。第一，它减少了模型把姿态和表情变化误认为身份差异的风险；第二，它使后续残差图可以在统一脸部语义坐标中平均和比较。论文后续的解释性分析依赖这一点：只有所有 residual map 都处在同一个 UV 空间中，跨视频、跨身份的像素级聚合才有明确语义（见 PAGE 5-6、PAGE 11）。

### 3. MAE 编码-解码与联合训练目标

CUPID 使用 ViT-based encoder-decoder 操作 UV texture map。输入 $I_{UV}$ 被划分为不重叠 patch，线性嵌入为 $C$ 维 token，加入可学习位置编码，并在序列前加入 CLS token。训练时，比例为 $\rho$ 的 patch 被随机遮挡，编码器只处理可见 token，解码器插入 mask token 后重建完整 UV texture map $\hat{I}_{UV}$（见 PAGE 3-4）。

论文的训练目标由三个部分组成：masked reconstruction loss、multi-layer contrastive loss 和 perceptual loss。总损失写为：

$$
L=\tau\cdot L_{REC}+(1-\tau)\cdot L_{CONT}+\lambda_p\cdot L_{PERC}
$$

其中 $L_{REC}$ 是遮挡重建损失，$L_{CONT}$ 是多层对比损失，$L_{PERC}$ 是感知损失；$\tau$ 控制重建目标与对比目标的权衡，$\lambda_p$ 控制感知损失权重。这个公式说明，CUPID 不只要求图像重建准确，还要求潜空间按身份聚类，并减少 MAE 重建常见的块状伪影（见 PAGE 4，公式 6）。

遮挡重建损失定义为：

$$
L_{REC}=\frac{1}{\rho}\sum_{p\in M}\left\|[\hat{I}_{UV}]_p-[I_{UV}]_p\right\|_F^2
$$

其中 $M$ 是被遮挡 patch 的集合，$p$ 表示某个 patch 位置，$[I_{UV}]_p$ 与 $[\hat{I}_{UV}]_p$ 分别是原始与重建 patch 的像素矩阵，$\|\cdot\|_F$ 是 Frobenius 范数。这个损失只在被遮挡区域计算，迫使模型从周围脸部上下文推断缺失纹理，而不是复制可见像素（见 PAGE 4，公式 7）。

对比损失用于增强身份判别性。设 $K$ 是选取的编码器中间层集合，$z_k\in\mathbb{R}^{C}$ 是第 $k$ 层的单位归一化 CLS embedding。对于 anchor 样本 $a$，$P(a)$ 表示 batch 中与 anchor 具有相同身份的其他样本，论文使用 NT-Xent 损失：

$$
L^{(k)}_{NT\text{-}Xent}
=
-\frac{1}{|P(a)|}
\sum_{j\in P(a)}
\log
\frac{
\exp(z^{(a)}_k\cdot z^{(j)}_k/t)
}{
\sum_{m\ne a}\exp(z^{(a)}_k\cdot z^{(m)}_k/t)
}
$$

其中 $t$ 是 temperature 参数。该公式的含义是：同一身份的 UV 纹理 embedding 应该彼此接近，不同身份的 embedding 应该拉开距离（见 PAGE 4，公式 8）。

多层对比损失进一步写为：

$$
L_{CONT}=\sum_{k\in K}w_k\cdot L^{(k)}_{NT\text{-}Xent}
$$

其中 $w_k$ 是第 $k$ 层的权重。论文选择多层 supervision 的原因是不同 Transformer 层捕获的信息层级不同，较深层通常更偏高层语义；将多个深度纳入对比训练可以提高身份表示稳定性（见 PAGE 4，公式 9）。

感知损失使用冻结的 VGG-16 中间特征，定义为：

$$
L_{PERC}=\frac{1}{|F|}\sum_{l\in F}\left\|\phi_l(\hat{I}_{UV})-\phi_l(I_{UV})\right\|_1
$$

其中 $F$ 是选定 VGG 层集合，$\phi_l(\cdot)$ 表示第 $l$ 层特征图。这个公式不是直接比较像素，而是比较预训练网络中的感知特征，目的是抑制单纯 MSE 重建带来的网格状 patch 边界伪影，使 UV 重建在视觉结构上更连贯（见 PAGE 4-5，公式 10）。

### 4. 推理阶段：Reference set 与 pairwise similarity

在部署阶段，CUPID 不再使用随机遮挡，因为遮挡会移除判别信息。对于某个测试 POI，系统假设拥有 $N_{vid}$ 个真实参考视频，并从每个参考视频采样 $N_{frm}$ 帧，因此参考样本总数为 $N_r=N_{vid}N_{frm}$。待测视频也采样 $N_t=N_{frm}$ 个 frame-level 样本（见 PAGE 5）。

设参考 token embedding 集合为 $R=\{R_i\}_{i=1}^{N_r}$，测试 token embedding 集合为 $T=\{T_i\}_{i=1}^{N_t}$，其中 $R_i,T_i\in\mathbb{R}^{T\times C}$，$T$ 是 token 数，$C$ 是 token 维度。每个矩阵的第一个 token 是 CLS embedding；参考 CLS 集合记为 $R_{CLS}=\{r_i\}_{i=1}^{N_r}$，测试 CLS 集合记为 $T_{CLS}=\{t_i\}_{i=1}^{N_t}$，其中 $r_i,t_i\in\mathbb{R}^{C}$（见 PAGE 5）。

CUPID 用余弦相似度比较第 $i$ 个参考 CLS embedding 与第 $j$ 个测试 CLS embedding：

$$
s_{ij}=
\frac{r_i^\top t_j}{\|r_i\|_2\|t_j\|_2},
\quad
i\in\{1,\ldots,N_r\},\;
j\in\{1,\ldots,N_t\}
$$

其中 $s_{ij}$ 越大，表示该参考帧和测试帧的身份纹理表示越接近。这个公式把视频级检测问题转化为参考帧集合与测试帧集合之间的 pairwise similarity 计算（见 PAGE 5，公式 11）。

最终分数定义为相似度矩阵 $S\in\mathbb{R}^{N_r\times N_t}$ 中的最大值：

$$
s^\star=\max_{i,j}[S]_{ij}
$$

这个设计背后的操作假设是：真实测试视频中至少应有部分帧与 POI 真实参考分布足够接近；伪造视频则应在整体上产生更低相似度。最大相似度聚合使系统关注“是否存在可靠真实匹配”，而不是要求每一帧都高度一致（见 PAGE 5，公式 12）。

### 5. 可解释性：从潜空间位移到 UV 残差图

CUPID 的可解释性来自两个条件：3DMM 提供的 UV 空间稠密语义对齐，以及 MAE 学到的可解码潜空间。论文将解释目标定义为：在 UV 空间中定位哪些面部区域最影响 fake 决策（见 PAGE 5）。Fig. 5 给出了这一分析流程的示意，包括 latent-space view、decoded displacement map 和 interpretability map（见 PAGE 5-6）。

首先，给定某个 POI 的参考 token embedding $R_i$，论文定义 reference centroid：

$$
\bar{R}=\frac{1}{N_r}\sum_{i=1}^{N_r}R_i
$$

其中 $\bar{R}$ 表示该 POI 在潜空间中的规范外观中心。这个平均操作用于抑制单个参考帧中的姿态、光照、表情残差，同时保留身份相关结构；它之所以在像素语义上有意义，是因为所有样本已经对齐到 UV 空间（见 PAGE 5，公式 13）。

对于测试视频，论文计算每个 reference-test pair 的 latent displacement：

$$
\delta^{(T)}_{ij}=T_j-R_i
$$

然后将该位移锚定到 subject centroid 上：

$$
Z^{(T)}_{ij}=\bar{R}+\delta^{(T)}_{ij}
$$

其中 $Z^{(T)}_{ij}$ 是 centroid-anchored test displacement。直观地说，这一步把“测试样本相对参考样本的差异”施加到该 POI 的平均身份原型上，以便后续通过 decoder 映射回 UV 空间（见 PAGE 6，公式 14）。

为了区分伪造导致的偏移和真实参考样本之间的自然变化，论文还定义 reference-reference displacement：

$$
Z^{(R)}_{ij}=\bar{R}+\delta^{(R)}_{ij},
\quad
\delta^{(R)}_{ij}=R_j-R_i,
\quad
i\ne j
$$

这个公式刻画了同一 POI 的真实参考视频内部本来就存在的变化，例如光照、表情或 3DMM 拟合残差。后续解释图会减去这部分 reference-only variability，以避免把正常变化误标为伪造证据（见 PAGE 6，公式 15）。

设 MAE decoder 为 $D:\mathbb{R}^{T\times C}\to\mathbb{R}^{H\times W}$，测试视频的 decoded displacement map 定义为：

$$
U^{(T)}
=
\frac{1}{|P_T|}
\sum_{(i,j)\in P_T}
D\left(Z^{(T)}_{ij}\right)
$$

其中 $P_T=\{1,\ldots,N_r\}\times\{1,\ldots,N_t\}$ 是所有 reference-test pair。这个公式把所有测试相关位移解码回 UV 图像空间并求平均，得到待测视频相对参考身份的空间偏移模式（见 PAGE 6，公式 16）。

参考内部 decoded displacement map 定义为：

$$
U^{(R)}
=
\frac{1}{|P_R|}
\sum_{(i,j)\in P_R}
D\left(Z^{(R)}_{ij}\right)
$$

其中 $P_R=\{1,\ldots,N_r\}\times\{1,\ldots,N_r\}, i\ne j$。它表示真实参考样本之间的正常空间变化模式，可视为 subject-specific normalizer（见 PAGE 6，公式 17）。

最终解释图为：

$$
M^{(T)}=\left|U^{(T)}-U^{(R)}\right|
$$

其中 $M^{(T)}$ 是测试视频的 interpretability map。这个图在 UV 空间中逐像素显示测试视频相对真实参考分布的异常残差；真实视频的残差应较小且分散，伪造视频则应在被篡改的人脸区域形成更集中的响应（见 PAGE 6，公式 18）。

### 6. 架构与训练配置

CUPID 的 UV 纹理提取 pipeline 使用 RetinaFace 进行人脸检测与对齐，使用 3DDFA-V3 进行 3D 人脸重建。UV texture map 分辨率设为 $224\times224\times3$，与训练集 VoxCeleb2 的帧分辨率和 ViT/MAE 常见设置一致（见 PAGE 7）。

模型采用 $16\times16$ 不重叠 patch，因此每张 $224\times224$ UV 图得到 $T=196$ 个 patch token。每个 patch 被投影到 $C=192$ 维 token。编码器包含 12 个 Transformer block，每个 block 有 3 个 attention head；解码器包含 4 个 Transformer block，同样使用 3 个 attention head，hidden dimension 为 192。重建头为每个 token 预测 $3\times16^2=768$ 维扁平 patch，对应 RGB 三通道的 $16\times16$ 像素（见 PAGE 7）。

训练超参数通过 Weights & Biases 的 Bayesian sweep 搜索选择。遮挡率 $\rho=0.5$，即训练时随机遮挡一半 patch；损失平衡参数 $\tau=0.5$，重建与对比目标权重相等；感知损失权重 $\lambda_p=0.1$。VGG-16 感知层使用 `{conv1_2, conv2_2, conv3_3, conv4_3}`，NT-Xent temperature 为 $t=0.1$，对比特征层为 $K=\{3,6,9,12\}$，层权重为 $(0.1,0.2,0.3,0.4)$（见 PAGE 8）。

训练优化器为 AdamW，$\beta_1=0.9$、$\beta_2=0.95$，基础学习率 $1\times10^{-4}$，weight decay 为 0.1。学习率采用 cosine annealing，并有 10 epoch linear warmup；梯度裁剪最大范数为 1.0；最多训练 100 epoch，验证集 10 epoch 无提升则 early stopping（见 PAGE 8）。每个 batch 包含 $N_{id}=8$ 个身份，每个身份采样 $N_{vid}=5$ 个视频，每个视频采样 $N_{frm}=5$ 帧，总 batch size 为 $B=8\times5\times5=200$ 张 UV texture map（见 PAGE 8）。

训练增强包括：以 0.3 概率应用 color jitter、Gaussian noise 或 Gaussian blur 之一；以 0.5 概率应用 JPEG compression，质量 $q\in[30,90]$。此外，每个 epoch 只随机使用 50% 身份，以促进泛化并加速训练（见 PAGE 8）。这些增强与采样策略共同服务于论文的核心目标：在真实视频上训练出对压缩和跨数据集变化更稳定的身份纹理表示。

## 实验分析

### 1. 数据集与评估设置

CUPID 训练使用 VoxCeleb2，这是一个大规模音视频语料库，包含 6,112 个身份、150,480 个视频和 1,128,246 条 utterance；训练 split 有 5,994 个身份，验证 split 有 118 个身份。为避免与 FakeAVCeleb 构造身份重叠，论文移除了 FakeAVCeleb 使用的 500 个身份，并对保留身份按每个身份 21 个视频进行均衡采样（见 PAGE 6）。

测试阶段使用四个带身份元数据的数据集：DF-TIMIT、FakeAVCeleb、KoDF 和 DeepSpeak。论文强调，POI 任务需要 subject identity metadata，而不只是 real/fake 标签（见 PAGE 6）。每个测试集都有 Higher Quality（HQ）原始版本和 Lower Quality（LQ）版本，LQ 是额外 H.264 压缩并设置 `crf40` 得到的版本；包含 LipSync 生成样本的数据集中，论文排除了 LipSync 视频，因为 LipSync 主要编辑嘴部而不改变身份线索，不符合 full-identity manipulation 的目标设定（见 PAGE 6）。

| 数据集 | 真实视频/主体信息 | 伪造视频/生成方式 | 分辨率或特点 | 证据 |
|---|---:|---:|---|---|
| DF-TIMIT | VidTIMIT 含 43 个 subjects，每个 13 个视频；实验涉及 32 个身份 | 320 个 fake，autoencoder-based bidirectional face-swap | 控制场景，$512\times384$ | PAGE 6 |
| FakeAVCeleb | 来自 VoxCeleb2 的 500 个真实 celebrity 视频 | 19,500 个 fake，含 FaceSwap、FSGAN、SV2TTS、Wav2Lip 等 | $224\times224$；本文只使用视频被篡改子集 | PAGE 7 |
| KoDF | 403 个 subjects，62,166 个 real clips | 175,776 个 fake clips，含 FaceSwap、DeepFaceLab、FSGAN、FOMM、ATFHP、Wav2Lip | $1920\times1080$，大规模韩语身份标注数据集 | PAGE 7 |
| DeepSpeak | 16,043 个 real videos，500 个 identities | 14,005 个 fake videos，含多种 recent generation tools | $1280\times720$；包含身份匹配和新型 avatar-style pipelines | PAGE 7 |

表格解读：这四个数据集覆盖了从早期受控 face-swap 到近年大规模身份匹配与 avatar-style 合成的不同伪造机制。CUPID 在这些数据集上评估，说明论文关注的不是单一数据集内拟合，而是跨生成管线、跨分辨率、跨压缩设置的 POI 身份一致性检测（见 PAGE 6-7）。

论文报告两个主要指标：ROC 曲线下面积 AUC 和 Balanced Accuracy（BA）。AUC 衡量不依赖阈值的排序质量，BA 衡量在阈值下的平衡分类准确率。论文同时报告 dataset-level 和 POI-level 两种设置：dataset-level 将所有 subjects 合并并使用一个全局阈值；POI-level 对每个身份单独计算 AUC/BA，再对身份求平均，允许每个 POI 单独校准阈值（见 PAGE 7）。这种双重设置很重要，因为真实部署既可能需要统一阈值，也可能允许对高价值 POI 做单独校准。

### 2. Reference set size：视频多样性比单视频多帧更关键

Fig. 6 分析了参考集大小对 POI-level AUC 的影响，横向比较 $N_{vid}$（参考视频数）和 $N_{frm}$（每个视频采样帧数），并在 HQ 与 LQ 两种质量设置下评估。DF-TIMIT 因性能在所有配置下接近 100% 而被省略（见 PAGE 8）。

结果显示，增加参考视频数量比增加单个视频内采样帧数更有效。以每个视频 1 帧为例，FakeAVCeleb 从 1 个参考视频增加到 10 个参考视频时，HQ 设置下 AUCpoi 提升 11 点，LQ 设置下提升超过 13 点；DeepSpeak 同样设置下，HQ 提升 7 点，LQ 接近 5 点；KoDF 也呈相同趋势但提升较小（见 PAGE 8）。这说明 POI 参考集应优先覆盖更多视频场景，而不是从同一视频中密集抽帧。

相反，增加每个视频的帧数带来的收益较弱且不完全单调。论文解释说，UV texture map 已经归一化了大量姿态和表情变化，因此同一视频内相邻或相似帧提供的信息可能冗余（见 PAGE 8-9）。最终，论文在后续实验中使用 10 个参考视频和每视频 10 帧。选择 $N_{frm}=10$ 不完全是为了检测性能，而是为了让可解释性分析中的 reference centroid、reference-reference displacement 和 reference-test displacement 更稳定（见 PAGE 9）。

### 3. 与 State of the Art 的 POI-level 比较

Table I 报告 POI-level metrics，即对每个身份分别计算 AUCpoi/BApoi 后再平均。下面摘录各方法在四个数据集上的均值，分 HQ 与 LQ 两组（见 PAGE 10，Table I）。

| 方法 | 类型 | HQ Mean AUCpoi / BApoi | LQ Mean AUCpoi / BApoi |
|---|---|---:|---:|
| RealForensics | 通用检测器 | 93.52 / 91.93 | 70.92 / 75.07 |
| LipForensics | 通用检测器 | 94.55 / 93.33 | 68.66 / 73.47 |
| FTCN | 通用检测器 | 88.21 / 88.90 | 59.22 / 68.90 |
| Seferbekov | 通用检测器 | 83.16 / 85.76 | 63.47 / 70.83 |
| poi-forensics | POI-specific | 80.86 / 82.73 | 79.87 / 81.69 |
| CUPID | POI-specific | 89.27 / 88.30 | 86.07 / 85.45 |

表格解读：在 HQ 设置下，LipForensics 的 mean AUCpoi 最高，为 94.55，CUPID 为 89.27，低 5.28 点；但在 LQ 设置下，CUPID 的 mean AUCpoi 为 86.07，成为表中最强方法。通用检测器在压缩后下降明显，例如 LipForensics 从 94.55 降到 68.66，而 CUPID 只从 89.27 降到 86.07。这个结果支撑论文关于压缩鲁棒性的核心主张：UV 身份纹理表征比直接依赖帧级伪造痕迹的通用检测器更能抵抗质量退化（见 PAGE 9-10）。

与 POI-specific baseline poi-forensics 相比，CUPID 在 HQ mean AUCpoi 上从 80.86 提升到 89.27，在 LQ mean AUCpoi 上从 79.87 提升到 86.07（见 PAGE 10）。这表明改进并不只是来自 POI 任务设定本身，而是来自 UV texture map、MAE identity representation 和相似度匹配的组合。

### 4. Dataset-level 比较：统一阈值下的稳定性

Table II 报告 dataset-level metrics，即将一个数据集内所有 subjects 合并，并使用单个阈值评估 AUC/BA。这个设置更严格，因为它不允许每个 POI 单独选择最佳阈值（见 PAGE 7、PAGE 10）。

| 方法 | 类型 | HQ Mean AUC / BA | LQ Mean AUC / BA |
|---|---|---:|---:|
| RealForensics | 通用检测器 | 91.56 / 85.49 | 68.36 / 64.07 |
| LipForensics | 通用检测器 | 92.07 / 85.66 | 66.11 / 62.71 |
| FTCN | 通用检测器 | 87.11 / 81.49 | 57.09 / 59.94 |
| Seferbekov | 通用检测器 | 80.57 / 77.01 | 62.73 / 61.36 |
| poi-forensics | POI-specific | 79.44 / 75.72 | 78.43 / 74.73 |
| CUPID | POI-specific | 87.94 / 83.02 | 84.20 / 79.53 |

表格解读：HQ dataset-level 下，LipForensics 的 mean AUC 为 92.07，CUPID 为 87.94；但 LQ dataset-level 下，CUPID 的 mean AUC 为 84.20，明显高于 poi-forensics 的 78.43，也高于所有通用检测器。dataset-level 的结果说明，在统一阈值部署场景中，CUPID 的压缩鲁棒性仍然成立，而不是只在 POI-level 个体校准下成立（见 PAGE 10）。

论文进一步用 BApoi 与 BA 的差距衡量阈值稳定性。通用检测器在 HQ 和 LQ 设置下平均差距分别为 7.57 和 10.05 点，而 CUPID 分别为 5.28 和 5.92 点；poi-forensics 的差距为 7.01 和 6.96 点（见 PAGE 9-10）。这说明 CUPID 在从 subject-specific calibration 转向 dataset-level shared threshold 时损失更小，阈值迁移更稳定。

### 5. KoDF 例外与分布偏移

尽管 CUPID 在多数数据集和条件下强于 poi-forensics，KoDF 是一个系统性例外。论文指出，poi-forensics 在 KoDF 上略强，作者推测原因可能是 KoDF 与训练数据的分布不匹配：KoDF 是以韩国 subjects 为主的大规模数据集，而 CUPID 训练使用的 VoxCeleb2 中相关族群可能代表性不足（见 PAGE 9）。论文将其与 Deepfake detection 中已知的 ethnicity-related distribution shift 联系起来（见 PAGE 9）。

这一点对业务部署很关键。CUPID 学习的是身份外观的潜空间分布，如果训练集中族群、肤色、拍摄条件或人脸形态覆盖不足，模型可能在某些人群上表现下降。论文在结论中也将更具族群多样性的训练数据列为未来改进方向（见 PAGE 12）。

### 6. 缩放鲁棒性与阈值稳定性

Fig. 7 报告了空间 downscaling sweep，resize factor 从 0.9 到 0.1。论文只比较 CUPID 与 poi-forensics，因为通用检测器对这类扰动已知敏感，且前文显示 POI 方法更稳健（见 PAGE 9-10）。结果显示 CUPID 在整个缩放范围内更平滑地退化，并持续优于 poi-forensics。KoDF 起初是 poi-forensics 更强，但当 resize factor 低于 0.3 后，CUPID 反超（见 PAGE 10）。

论文特别解释，KoDF 原始分辨率为 $1920\times1080$，因此 resize factor 0.2 仍对应 $384\times216$，并非完全不现实的极低质量，而是类似社交媒体二次压缩后可能出现的分辨率（见 PAGE 10）。这支撑了 CUPID 对在线传播视频有实际意义的鲁棒性主张。

Fig. 8 分析了每个 subject 上最大化 BApoi 的阈值分布。CUPID 在 FakeAVCeleb、DeepSpeak 和 KoDF 上的阈值分布非常集中，标准差范围为 0.014 到 0.042；poi-forensics 则呈长尾分布，在 FakeAVCeleb 上极端值达到 6.4，在 KoDF 上达到 13.4（见 PAGE 10-11）。这说明 CUPID 的分数尺度更稳定，部署时对每个 POI 单独阈值校准的依赖更弱。

### 7. 计算效率

论文在 DeepSpeak 的 50 个 subjects、共 2092 个视频上统计 inference time，结果见 Table III（见 PAGE 11）。

| 方法 | Mean time (s) | Std time (s) |
|---|---:|---:|
| CUPID | 0.155 | 0.043 |
| poi-forensics | 4.961 | 5.141 |

表格解读：CUPID 平均每视频 0.155 秒，poi-forensics 平均每视频 4.961 秒，论文称 CUPID 快约 $32\times$（见 PAGE 11）。速度差异来自处理策略：CUPID 每个视频固定采样 $N_{frm}=10$ 帧并对每帧运行一次编码器，因此 forward pass 数量不随视频长度增长；poi-forensics 使用 sliding-window 策略，需要处理覆盖整段视频的多个重叠 temporal segment，因此推理时间均值和标准差都更高（见 PAGE 11）。

### 8. 可解释性实验

Fig. 9 展示了四个数据集上的 interpretability residual maps。四列分别是随机选中身份的 real map、fake map，以及整个数据集所有身份平均后的 real map、fake map；亮度越高表示相对 POI 参考身份的偏离越大（见 PAGE 11-12）。

论文观察到，在所有数据集中，fake videos 的 interpretability map 比 real videos 激活更强。DF-TIMIT 的 fake residual 主要集中在眼睛、鼻子和脸颊，符合 autoencoder-based face-swap 的局部伪造特点；KoDF 还出现嘴部强激活，可能与其 face-swapping 与 reenactment 结合有关；FakeAVCeleb 的响应更靠近外脸边界，可能对应 face-swap blending artifacts；DeepSpeak 的激活更弥散，与其包含更多 recent generation 与 avatar-style pipelines 相符（见 PAGE 11）。

数据集平均图进一步支持同一结论：real maps 整体较低，fake maps 较高；DF-TIMIT 和 KoDF 更局部，FakeAVCeleb 和 DeepSpeak 更弥散（见 PAGE 11-12）。不过论文也指出，FakeAVCeleb 的 real residual 相对偏高，KoDF 与 DeepSpeak 的 fake residual 相对偏低；作者推测后两者可能因为生成质量更高而更难暴露（见 PAGE 12）。因此，解释图的实用重点不是跨数据集比较绝对残差大小，而是同一 POI 内 real/fake residual pattern 的相对差异（见 PAGE 12）。

## 讨论

CUPID 适用于拥有目标人物真实参考视频的检测场景。它不适合没有明确 POI、没有参考视频、或目标是对任意陌生用户做开放集 Deepfake 分类的场景。论文的推理阶段明确要求有该 POI 的 pristine reference support，并基于参考集合与待测集合的 embedding 相似度进行真伪判断（见 PAGE 5）。因此，CUPID 更像身份冒用风险打分器，而不是通用内容安全分类器。

从方法论看，CUPID 的核心贡献不是单独提出 MAE，也不是单独使用 UV texture map，而是将两者组织为一个 POI 检测流程：UV 空间提供跨帧、跨身份的语义对齐；MAE 提供真实人脸纹理的上下文重建与身份表示；contrastive loss 强化身份聚类；reference-test similarity 将训练得到的通用表示转化为目标人物真伪检测；decoded residual map 提供解释（见 PAGE 2-6）。这条链路的每个环节都服务于同一个目标：在不训练目标 POI 专属模型的情况下，判断待测视频是否偏离该 POI 的真实身份纹理分布。

与通用 Deepfake 检测器相比，CUPID 的优势主要出现在低质量视频、缩放压缩和阈值稳定性上。Table I 和 Table II 表明，HQ 下最强通用检测器仍可能优于 CUPID，但 LQ 下 CUPID 的 mean AUC/AUCpoi 最高（见 PAGE 10）。这说明 CUPID 可能不是在所有无扰动 benchmark 上取得最高分的模型，但它更贴近在线传播场景中视频被转码压缩后的部署需求。

业务上，CUPID 对“目标人物保护”有清晰价值。若系统已拥有某个高风险人物的采访、发布会、直播回放等干净素材，则可以构建参考集，用 CUPID 对新出现的视频做身份一致性检测，并通过 UV residual map 辅助人工复核。但如果参考视频存在压缩严重、光照单一、妆容变化大、遮挡多或 3D 重建失败等问题，reference distribution 本身可能不稳定，这一点论文没有通过专门实验充分展开，属于部署前必须验证的工程风险。

## 局限分析

第一，作者自述的局限是当前 CUPID 是 video-only 且 identity-centric 的方法，没有显式建模 temporal expressivity，因此对 LipSync 类只改嘴部运动、身份线索保持不变的 manipulation 并不适配。论文在测试设置中排除了 LipSync 样本，并在结论中提出未来要加入 subject-specific dynamics，以更好处理 LipSync manipulation（见 PAGE 6、PAGE 12）。这意味着 CUPID 不能被直接视为完整的音视频伪造检测方案。

第二，作者自述的另一局限是 KoDF 上的较弱结果显示模型可能对 ethnicity-related distribution shift 敏感。论文把这一现象归因于 KoDF 中韩国 subjects 与 VoxCeleb2 训练分布之间可能存在族群代表性差异，并将更具族群多样性的训练数据列为未来方向（见 PAGE 9、PAGE 12）。对于实际部署，这对应公平性和跨人群泛化问题，不能只用总体 AUC 掩盖。

第三，本文的独立判断是，CUPID 强依赖人脸检测、对齐和 3D face reconstruction 的质量。论文使用 RetinaFace 和 3DDFA-V3，并将 UV extraction 作为整个 pipeline 的前置步骤（见 PAGE 7）。如果视频存在大姿态、遮挡、极端光照、低分辨率、运动模糊或非正脸，UV texture map 可能出现拟合误差；而后续 MAE 和 residual map 都建立在 UV 对齐正确的假设上。论文虽然评估了压缩和缩放鲁棒性，但没有在提供文本中系统报告 3D 重建失败率或对齐误差对最终检测的影响，证据不足。

第四，最大相似度聚合 $s^\star=\max_{i,j}[S]_{ij}$ 具有明确效率优势，但也可能引入边界情况：如果伪造视频中存在少数帧与参考帧高度相似，最大值可能抬高整体真值分数；反过来，如果真实视频因妆容、遮挡、年龄变化或参考集覆盖不足而缺少高相似帧，也可能被压低（见 PAGE 5）。论文用实验显示整体效果良好，但提供文本中没有单独给出 max pooling 与 mean pooling、top-k pooling 等聚合策略的消融对比，证据不足。

第五，代码可复现性目前不足。论文说明 experimental code will be released at `polimi-ispl/CUPID`，但当前材料没有可确认源码、README、配置或训练脚本（见 PAGE 1）。因此本文不能验证 RetinaFace/3DDFA-V3 调用细节、数据清洗脚本、训练采样实现、阈值选择过程或 interpretability map 的具体数值处理。所有源码级分析均证据不足。

## 结论

CUPID 提出了一条面向 POI Deepfake 检测的结构化路线：先用 3DMM 将人脸外观映射到 UV texture map，再用 MAE 在真实视频上学习身份一致的潜表示，最后在推理阶段将待测视频与目标 POI 的真实参考视频做 embedding 相似度匹配，并通过 UV decoded residual map 输出解释（见 PAGE 1-6）。它的关键价值在于同时追求三点：不需要伪造样本训练、不需要目标 POI 参与训练、能够在低质量视频下保持较好鲁棒性。

实验上，CUPID 在 HQ 场景不总是超过最强通用检测器，但在 LQ 场景取得最好的 mean AUC/AUCpoi，并且比公开 POI baseline poi-forensics 更快、更稳定；在 DeepSpeak 2092 个视频上的平均推理时间为 0.155 秒，相比 poi-forensics 的 4.961 秒快约 32 倍（见 PAGE 10-11）。这些结果使 CUPID 对有参考视频的高价值人物冒用检测具有现实参考意义。

同时，CUPID 的适用边界也很清楚：它依赖干净 POI 参考视频、可靠 UV 重建和训练分布覆盖；对 LipSync、族群分布偏移、3D 重建失败和源码复现仍存在未解决问题（见 PAGE 6-7、PAGE 9、PAGE 12）。因此，若用于业务试验，合理路径应是先在已有目标人物参考视频上进行小规模验证，重点观察压缩后表现、阈值稳定性、UV residual map 是否符合人工复核直觉，以及不同人群和拍摄条件下的误报/漏报差异。

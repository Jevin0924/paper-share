本文不是提出一个新的 AI 生成图像检测器，而是对训练无关检测方法（training-free detectors）做一次受控审计。论文把 AEROBLADE-style 自编码器重建分数、RIGID-style 噪声扰动特征相似度分数，以及一个朴素 feature-kNN control 放到同一个 1,500 张 GenImage-derived benchmark 上，在 LPIPS backbone、预处理分辨率、噪声强度、特征层级、JPEG 重编码等因素上逐一控制变量。核心结论是：训练无关检测器的性能数字高度脆弱，分数方向、预处理策略和数据格式来源都可能把看似稳健的 AUROC 结论改写甚至反转，见 PAGE 1、PAGE 2、PAGE 3。

一句话概括：本文证明训练无关 AI 生成图像检测分数在“怎么预处理、分数朝哪边、数据是否统一重编码”这些工程细节上高度敏感，因此上线评测不能只看 pooled AUROC，而必须按生成器族群、分数方向和数据来源逐项审计。

本文的业务价值在于，它把检测系统上线验收中容易被忽略的三类风险具体化：第一，LPIPS backbone 从 AlexNet 换到 VGG-16，clean overall AUROC 从 0.740 提升到 0.825，差值 +0.085，足以超过很多论文中方法间的差距，见 Table 1 / PAGE 2。第二，RIGID-style 分数在 $\sigma=0.05$ 时对 SD1.5 与 Wukong 低于随机水平，在 $\sigma=0.01$ 时又恢复到高于 0.5，说明“分数越大越像真图”并不是方法固有属性，而是超参数相关属性，见 Table 3 / PAGE 2。第三，若不统一 real 与 fake 的 JPEG provenance，JPEG-50 反而可能让 AEROBLADE-style AlexNet 分数看起来比 clean 更强，属于数据格式偏差造成的 robustness artifact，见 Table 1、Table 5 / PAGE 2-3。

## 背景与动机

AI 生成图像检测（AI-generated image detection）的基本任务，是判断输入图像来自真实摄影分布还是生成模型分布。随着 GAN、扩散模型、文本条件生成系统的发展，合成图像成为取证、内容安全和平台审核中的重要对象。论文在 Introduction 中把已有检测路线分成若干类：训练式分类器、生成器指纹与频域 artifact 检测、基于 CLIP 等预训练表示的近邻或线性探测方法，以及近年兴起的 training-free score 方法，见 PAGE 1。

传统训练式方法通常用 real/fake 图像对训练 CNN 或其他分类器。它们的优点是可以直接优化分类目标，缺点是容易受压缩、缩放、未知生成架构和数据集偏差影响。论文引用早期 CNN-generated image detection 工作与后续 critical analysis，指出这类检测器在 compression、resizing、unseen architectures 下会退化，见 PAGE 1。这一点对于实际部署尤其关键，因为线上图片往往经历平台压缩、裁剪、转码与多轮传播。

第二类方法利用生成器留下的指纹，例如上采样带来的频率 artifact。论文指出 GAN 到 diffusion model 的合成图像仍可能保留某些 attenuated artifacts，但这些 artifact 会因模型架构、后处理和压缩而变化，见 PAGE 1。因此，依赖某种固定 fingerprint 的检测器，未必能跨生成器稳定工作。

第三类方法基于大规模预训练视觉表示，例如 CLIP、DINOv2 或其他 foundation model representation。它们可以用 nearest-neighbor、linear probing 或专家混合等方式做检测。本文纳入的 kNN control 就属于这种非重建、非扰动的朴素基线：用 DINOv2 final CLS features 计算到 held-out real images 的近邻距离，大距离表示更可能是假图，见 PAGE 2。

本文关注的是第四类：训练无关检测器（training-free detectors）。这类方法不训练新的分类器，而是从冻结模型中计算一个分数，再用阈值判断真伪。论文列举的代表包括 latent-diffusion autoencoder reconstruction error、self-supervised features 对噪声扰动的 sensitivity、diffusion reconstruction error、高频 aliasing、edit-induced reconstruction shift，以及 lossless model coding cost 等，见 PAGE 1。它们的吸引力在于部署简单、看似 generator-agnostic，并且不需要为每个新生成器重新训练分类器。

问题在于，training-free 并不等于 evaluation-free。论文指出，已有文献常在异质协议下报告结果：不同 real/fake source、不同分辨率和 crop policy、不同 perceptual metric implementation，以及常常未充分报告的 robustness pipeline，见 PAGE 1。若一个方法在自己的数据管线中 AUROC 很高，但换 backbone、换 resize 策略或统一 JPEG provenance 后结论发生变化，那么论文数字就不能直接转化为上线验收标准。

本文的动机因此不是“再造一个检测器”，而是“审计检测分数的可解释稳定性”。作者明确说本文贡献是 deliberately small, fully controlled audit rather than a new detector：重新实现两个代表性 training-free scores 与一个朴素 control，冻结单一评测集，然后一次只改变一个因素，包括 perceptual-metric backbone、preprocessing resolution policy、perturbation strength、feature depth，以及 dataset JPEG re-encoding，见 PAGE 1。这种设计的意义在于把“方法差异”与“实现细节差异”分离开来。

## 预备知识

首先是 AUROC。论文所有主要结果都报告 threshold-free AUROC，并以 fake 作为 positive class，见 PAGE 1。若记检测分数为 $s(x)$，真实标签 $y=1$ 表示 fake，$y=0$ 表示 real，则 AUROC 衡量的是随机抽取一张 fake 与一张 real 时，fake 分数排在 real 前面的概率。符号化可写为：

$$
\mathrm{AUROC}=\Pr(s(x_{\mathrm{fake}})>s(x_{\mathrm{real}})).
$$

这句话的含义是：AUROC 不依赖某一个固定阈值，而依赖分数排序。若分数方向错了，即 fake 的分数系统性低于 real，那么 AUROC 会低于 0.5；这种情况不是“性能差一点”，而是检测符号方向发生了反转。本文最重要的警示之一正是 RIGID-style score 在部分生成器和噪声强度下出现 AUROC < 0.5，见 Table 3 / PAGE 2。

其次是 score direction。训练无关方法通常隐含一个方向假设。例如 AEROBLADE-style 假设某些生成图像更容易被 Stable Diffusion VAE 重建，因此重建误差更低；RIGID-style 假设真实图像对噪声扰动更 robust，因此原图与扰动图在特征空间中的 cosine similarity 更高，见 PAGE 2。问题在于，这些假设不一定对所有生成器和所有超参数成立。

第三是 dataset format bias。GenImage 中 real images 是 JPEG-sourced，而 fakes 是 PNG，论文指出这是一种 known confound，见 PAGE 2-3。如果原始 real/fake 的编码来源不同，那么再做 JPEG-70 或 JPEG-50 degradation 时，真实图和假图并不是从同一种 provenance 出发。此时 robustness trend 可能混入“格式来源差异”，而不是检测器本身对压缩的鲁棒性。

第四是 per-generator evaluation。论文每个 per-generator cell 使用该生成器的 100 张 fakes 对比全部 800 张 reals，见 PAGE 1。这种设定让作者能够观察 pooled AUROC 掩盖的异质性。例如 RIGID-style pooled clean AUROC 为 0.692，但 per-generator 从 SD1.5 的 0.408 到 BigGAN 的 0.931 差异极大，见 Table 2 / PAGE 2。对于内容安全团队，这意味着“总体能用”不代表“面对某个生成器族群能用”。

## 方法详解

### 受控 benchmark：固定评测集，再逐项改变变量

本文从 GenImage repackaging 的 validation portion 中抽样 1,500 张图像：800 张 real images，来自 ImageNet；700 张 fakes，来自七个生成器，每个生成器 100 张，包括 ADM、BigGAN、GLIDE、Midjourney、SD1.5、VQDM、Wukong。抽样使用固定 seed 42，见 PAGE 1。论文明确说明 validation split 包含 GenImage 八个生成器中的七个，不含 SD1.4，见 PAGE 1 脚注。

这一评测集可以符号化为：

$$
\mathcal{D}=\mathcal{D}_{\mathrm{real}}\cup \bigcup_{g\in \mathcal{G}}\mathcal{D}_{g},
\quad |\mathcal{D}_{\mathrm{real}}|=800,\quad |\mathcal{D}_{g}|=100,\quad |\mathcal{G}|=7.
$$

这个公式只是把 PAGE 1 的 benchmark 描述写成集合形式。它强调本文不是在大规模样本上追求绝对性能，而是在一个固定小规模审计集上最大化变量控制能力。

压缩鲁棒性通过 Pillow 把每张图像重编码到 JPEG quality $q=70$ 与 $q=50$ 来测量，见 PAGE 1。这里 $q$ 表示 JPEG quality 参数，数值越低通常压缩越强。论文还设置 bias-corrected pipeline：先把全部图像统一重编码到 JPEG $q=95$，再在此基础上做 $q=70/50$ degradation，见 PAGE 2。这一步是为了让 real 与 fake 具有相同 final compression provenance。

### Score 1：AEROBLADE-style autoencoder reconstruction score

第一个被审计的分数来自 AEROBLADE-style 思路。输入图像 $x$ 经过 Stable Diffusion VAE（论文指定为 `sd-vae-ft-mse`）编码再解码，得到重建图 $\hat{x}$，再计算原图和重建图之间的 LPIPS perceptual distance，见 PAGE 1-2。论文文字说明 score 是 perceptual distance $d_{\mathrm{LPIPS}}(x,\hat{x})$，并指出 fakes reconstruct better，也就是距离更低，见 PAGE 2。

可写为：

$$
s_{\mathrm{AERO}}(x)=d_{\mathrm{LPIPS}}(x,\hat{x}),\quad \hat{x}=\mathrm{VAE}_{\mathrm{dec}}(\mathrm{VAE}_{\mathrm{enc}}(x)).
$$

人话解释：这个分数衡量图像经过 Stable Diffusion VAE 压缩再还原后，与原图在感知特征上差了多少。如果某类生成图像本来就更贴近该 VAE 的重建偏好，它可能获得更低重建误差。

该分数存在一个方向问题：论文以 fake 为 positive class，但 AEROBLADE-style 文字假设是 fakes reconstruct better，即 fake 的 LPIPS distance 更低，见 PAGE 2。因此在计算 AUROC 时，实际实现必须处理“低分更像 fake”的方向，否则 AUROC 方向会错。论文没有展开给出符号公式，但它反复强调 score direction 的脆弱性，见 PAGE 1-3。

本文审计了 AEROBLADE-style 的两个实现因素。第一是 LPIPS backbone：AlexNet 与 VGG-16，其中原 AEROBLADE paper 使用 VGG，见 PAGE 2。第二是 preprocessing policy：resize-512，即 bicubic short-side resize to 512 + center crop；以及 native，即 native-resolution center crop 到 16 的倍数，并 capped at 512，见 PAGE 2。论文结果显示，仅 LPIPS backbone 变化就让 overall clean AUROC 从 0.740 变为 0.825，见 Table 1 / PAGE 2。

这一设计揭示的不是 VGG 一定“更好”，而是 leaderboard 或论文比较若混用 LPIPS implementation，可能比较的是 backbone，而不是检测思想。更严重的是，preprocessing policy 不只是改变平均数，还会改变 per-generator ranking。BigGAN 在 VGG backbone 下从 resize-512 的 0.911 掉到 native 的 0.528，而 Midjourney 反向从 0.834 升到 0.989，见 Table 2 / PAGE 2。

### Score 2：RIGID-style perturbation sensitivity score

第二个分数来自 RIGID-style 噪声扰动特征相似度。论文做法是对图像 $x$ 和噪声扰动图像 $x+\epsilon$ 分别计算 DINOv2-base embeddings，再用 cosine similarity 作为 score；噪声 $\epsilon$ 服从均值为 0、方差为 $\sigma^2I$ 的高斯分布，并对 3 次 noise draws 取平均，见 PAGE 2。输入图像被 bicubic resize 并 center-crop 到 $224^2$，见 PAGE 2。

论文中的噪声模型可写为：

$$
\epsilon\sim \mathcal{N}(0,\sigma^2 I).
$$

这句话的含义是：每个像素维度加入强度由 $\sigma$ 控制的独立高斯噪声，其中 $I$ 是单位矩阵，表示各维噪声协方差相互独立。

若记 DINOv2 embedding 为 $f_\ell(\cdot)$，其中 $\ell$ 表示 feature depth，则 RIGID-style 分数可符号化为：

$$
s_{\mathrm{RIGID}}(x)=\frac{1}{3}\sum_{r=1}^{3}
\cos\left(f_\ell(x), f_\ell(x+\epsilon_r)\right).
$$

人话解释：这个分数衡量同一张图在加入小噪声前后，DINOv2 特征是否稳定。如果相似度高，说明图像在该特征空间中对噪声更稳。

论文审计的超参数网格是：

$$
\sigma\in\{0.01,0.05,0.1,0.3\},\quad 
\ell\in\{\mathrm{layer\text{-}6\ CLS},\mathrm{final\ CLS}\}.
$$

这句话的含义是：作者同时改变噪声强度和特征层级，观察“真实图更 robust”这一方向假设是否稳定。结果表明并不稳定：$\sigma=0.05$ 时 SD1.5 与 Wukong 低于随机水平，$\sigma=0.01$ 时所有生成器恢复到高于 0.5，而 $\sigma=0.3$ final features 下整体 ALL AUROC 只有 0.361，SD1.5 低至 0.149，见 Table 3 / PAGE 2。

这里最关键的学术贡献是把 RIGID-style 的失败解释为 data property，而不是实现 bug。论文在 Section 3.3 中说明，score 是同一个 raw cosine-similarity column，对所有图像用同一公式计算；如果是 sign error，应当所有生成器统一翻转，而不是只翻转 SD1.5 与 Wukong，见 PAGE 3。Table 4 进一步显示，在 $\sigma=0.05$ 时 SD-family fake mean similarity 分别为 0.9568 与 0.9531，高于 real mean 0.9465，说明它们确实在 DINOv2 final-layer 特征中比真实图更噪声稳健，见 PAGE 3。

### Score 3：feature-kNN control

第三个分数是朴素 feature-kNN control，不属于重建方法，也不属于扰动敏感性方法。论文用 DINOv2 final CLS features，把每张测试图与 1,000 张 held-out real images 比较，计算到 $k=5$ 个最近邻的 mean cosine distance，距离越大越像 fake，见 PAGE 2。

可写为：

$$
s_{\mathrm{kNN}}(x)=\frac{1}{k}\sum_{i=1}^{k} d_{\cos}\left(f(x), f(x_i^{\mathrm{real}})\right),\quad k=5.
$$

人话解释：如果一张图在 DINOv2 特征空间中离真实图近邻集合很远，就更可疑；如果离真实图很近，则更像 real。

该 control 的意义不是追求最好结果，而是检验 reconstruction 与 perturbation scores 是否确实携带了比“离真实图远近”更多的信号。Table 1 显示 kNN control 在 clean/JPEG-70/JPEG-50 下分别为 0.568、0.558、0.557，接近随机但略高于 0.5，见 PAGE 2。论文据此认为 reconstruction 与 perturbation scores 的信号不能简单归结为 feature-space proximity to real images。

### Format-bias correction：统一 JPEG provenance

论文指出 GenImage 中 real images 是 JPEG-sourced，而 fakes 是 PNG，这是生成图像检测数据集中的已知混杂因素，见 PAGE 2-3。为校正这种格式偏差，作者把所有图像先统一 re-encode 到 JPEG $q=95$，然后再施加 $q=70/50$ degradation，见 PAGE 2。

可写为：

$$
x_{q95}=\mathrm{JPEG}_{95}(x),\quad 
x_{q}=\mathrm{JPEG}_{q}(x_{q95}),\quad q\in\{70,50\}.
$$

人话解释：不管原图来自 JPEG 还是 PNG，都先进入同一种轻压缩状态，再从同一个起点测试更强压缩的影响。这样 robustness trend 更可能反映检测器对压缩的响应，而不是原始数据格式差异。

这一校正是本文对业务评测最直接的贡献。原始 pipeline 中 AlexNet reconstruction score 在 JPEG-50 下从 clean 0.740 升到 0.762，看起来“压缩帮助检测”，见 Table 1 / PAGE 2。论文指出这显然可疑，因为压缩通常会破坏取证线索。经过 q95 统一重编码后，VGG variant 单调下降 0.835 → 0.790 → 0.775，而 AlexNet 的上升趋势缩小并定位到 BigGAN，见 Table 5 / PAGE 3。

### Fusion：互补失败集不等于简单平均有效

论文还测试了 z-score averaging fusion。由于 Table 2 显示 AEROBLADE-style 与 RIGID-style 有互补失败集，直觉上融合可能提升整体性能：RIGID 在 ADM、BigGAN、VQDM 上强，重建分数在 GLIDE、Midjourney 上强，见 PAGE 2。然而 Table 5 显示，所有四个分数 z-score fusion 在 q95/JPEG-70/JPEG-50 下为 0.782/0.757/0.753；最强 pair vgg+RIGID 为 0.829/0.783/0.772，仍不超过最佳单分数，见 PAGE 3。

原因在于 complementarity 需要 direction-aware combination。若 RIGID-style 在 SD-family 上方向错误，把它简单平均进融合分数，会给 SD1.5 与 Wukong 增加反向证据。论文在 Section 3.5 明确指出，naive fusion 不胜过最佳单分数，说明必须做 per-family sign-calibrated 或其他 direction-aware combination，见 PAGE 3。

## 实验分析

### 整体结果：backbone 与 preprocessing 足以支配 headline number

| Score Variant | clean | JPEG-70 | JPEG-50 |
|---|---:|---:|---:|
| AEROBLADE-style alex, resize-512 | 0.740 | 0.757 | 0.762 |
| AEROBLADE-style vgg, resize-512 | 0.825 | 0.794 | 0.776 |
| AEROBLADE-style alex, native | 0.735 | 0.708 | 0.693 |
| AEROBLADE-style vgg, native | 0.785 | 0.725 | 0.694 |
| RIGID-style $\sigma=0.05$, last | 0.692 | 0.673 | 0.677 |
| kNN control $k=5$, last | 0.568 | 0.558 | 0.557 |
| Fusion z-score alex-512 + RIGID | 0.762 | 0.754 | 0.758 |

表格解读：这张表复现论文 Table 1 的核心数据，见 PAGE 2。第一，LPIPS backbone 从 AlexNet 换到 VGG-16，在 resize-512 clean 条件下把 AUROC 从 0.740 提升到 0.825，差值 +0.085。这个差异来自同一方法内部的一行实现选择，因此若两个系统只报告“AEROBLADE-style”而不报告 LPIPS backbone，比较结论是不充分的。第二，preprocessing 改变了压缩趋势：AlexNet resize-512 在 JPEG 下反而上升，而 AlexNet native 单调下降。换言之，“JPEG 是否伤害检测器”不是检测器单独的属性，而是检测器、预处理和数据 provenance 共同决定的观测结果。

VGG resize-512 在 clean 条件下整体最佳，为 0.825；但在 JPEG-50 下只有 0.776。Alex native 与 VGG native 在 JPEG-50 下都降到 0.693/0.694。RIGID-style 在整体 pooled 视角下为 0.692，似乎只是中等性能；但这个 pooled number 后面掩盖了强烈的生成器异质性，见 Table 2 / PAGE 2。

### Per-generator 分析：同一方法在不同生成器上不是同一种检测器

| Generator | AERO alex-512 | AERO alex-native | AERO vgg-512 | AERO vgg-native | RIGID $\sigma=.05$ |
|---|---:|---:|---:|---:|---:|
| ADM | 0.714 | 0.663 | 0.772 | 0.697 | 0.816 |
| BigGAN | 0.812 | 0.605 | 0.911 | 0.528 | 0.931 |
| GLIDE | 0.956 | 0.885 | 0.983 | 0.922 | 0.822 |
| Midjourney | 0.804 | 0.954 | 0.834 | 0.989 | 0.587 |
| SD1.5 | 0.647 | 0.706 | 0.807 | 0.840 | 0.408 |
| VQDM | 0.626 | 0.650 | 0.649 | 0.664 | 0.834 |
| Wukong | 0.624 | 0.685 | 0.822 | 0.853 | 0.445 |

表格解读：这张表对应论文 Table 2，见 PAGE 2。最重要的不是哪一列最大，而是不同方法的失败集互补。RIGID-style 在 ADM、BigGAN、VQDM 上显著强于重建分数，尤其 BigGAN 为 0.931；但在 SD1.5 与 Wukong 上分别只有 0.408 和 0.445，低于随机水平。重建分数则在 GLIDE、Midjourney 上特别强，例如 VGG-native 对 Midjourney 达到 0.989。部署时如果只看 pooled AUROC，会把这些 generator-family-specific 风险抹平。

Preprocessing-induced swing 是这张表的第二个重点。BigGAN 在 VGG backbone 下从 resize-512 的 0.911 降到 native 的 0.528，幅度 0.383；Midjourney 则从 0.834 升到 0.989，方向相反，见 PAGE 2。论文据此指出，在一个 preprocessing policy 下得到的 cross-generator ranking 不能直接迁移到另一个 preprocessing policy。

表格解读：这张表对应论文 Table 4，见 PAGE 3。它用 raw cosine similarity 解释了 AUROC 反转的来源。$\sigma=0.05$ 时，real mean 为 0.9465，而 SD1.5 与 Wukong fake mean 分别为 0.9568 与 0.9531，高于 real；因此如果 score 假设“相似度越高越像 real”，这些 fake 会被排到 real 一侧，AUROC 自然低于 0.5。$\sigma=0.01$ 时所有 fake mean 都低于 real mean，方向恢复；$\sigma=0.3$ 时 SD-family fake 的相似度远高于 real，反转更强。论文还报告两次独立 scoring runs 在 1,500 张图像上的 Pearson $r=0.989$，且独立 Mann-Whitney implementation 复现所有 AUROC，说明该现象可重复，见 PAGE 3。

这里的关键不是 SD1.5/Wukong “更真实”，而是它们在 DINOv2 final-layer feature 下对某些噪声强度更稳定。论文在 Discussion 中给出一种机制性假设：SD-family latent models 的 VAE-decoded outputs 更局部平滑，当 $\sigma$ 超过自然 sensor noise 尺度后，其 embedding 比真实 ImageNet photos 更 noise-robust，见 PAGE 4。作者也谨慎说明这是与测量一致的 reading，而不是被实验完全证明的机制结论，见 PAGE 4。

## 讨论

本文最直接的结论是：pooled AUROC 对 training-free scores 接近无意义，至少不能单独作为上线验收依据。论文在 Discussion 中明确指出，pooled RIGID number 0.692 平均了从 0.408 到 0.931 的 per-generator values，见 PAGE 4。对于实际系统，风险并不按 pooled mean 发生，而是按攻击者或用户实际使用的生成器族群发生；如果线上图片主要来自 SD-family，那么一个在 BigGAN 上表现强的 RIGID-style 配置反而可能带来反向排序风险。

本文对生成器异质性的解释保持谨慎。RIGID-style 在 ADM、BigGAN、VQDM 上强，论文认为这些 pixel-space diffusion、GAN、vector-quantized models 可能具有更明显的 upsampling 与 spectral artifacts，小噪声会扰动这些高频 fingerprint，使 fake 在 feature space 中更不稳定，见 PAGE 3-4。相反，SD1.5 与 Wukong 作为 SD-family latent models，其 VAE-decoded outputs 可能更局部平滑，因此在某些噪声尺度下比真实 ImageNet photos 更 noise-robust，见 PAGE 4。但作者没有把这个解释当作已证明机制，尤其 Midjourney 与 Wukong 架构并不完全公开，见 Limitations / PAGE 4。

对评测规范而言，论文给出四点建议。第一，必须报告 per-generator results，pooled number 只能作为摘要，见 PAGE 4。第二，部署前必须验证每个 score 在目标生成器族群上的 sign，即分数方向，而不是默认“该方法方向固定”，见 PAGE 4。第三，默认进行 provenance-controlled re-encoding，并在两种 pipeline 结论不一致时同时报告，见 PAGE 4。第四，报告足够细的 per-generator 和 per-condition breakdown，让读者能在不同协议下复核结论，见 PAGE 4。

从方法学角度看，本文把 training-free detector 从“是否需要训练分类器”重新拉回到“是否需要校准与审计”。即使没有训练参数，检测分数也有 backbone、resolution policy、feature depth、noise strength、encoding provenance 等隐含配置。只要这些配置会改变分数排序或方向，系统就需要报告、验证和验收。换言之，training-free 不等于 assumption-free。

从业务角度看，本文尤其适合检测/内容安全团队作为评测 checklist。上线前至少需要回答以下问题：目标系统是否按生成器族群报告 AUROC？是否验证 score sign？是否统一 JPEG provenance？是否在 clean、JPEG-70、JPEG-50 或业务真实转码链路上分别评估？是否检查过简单融合会不会把某些生成器族群上的反向分数平均进去？这些问题都由本文的实验给出直接证据，见 PAGE 2-4。

## 局限分析

作者自述的局限较明确。第一，审计只覆盖一个 benchmark family，即 GenImage with ImageNet reals；七个生成器；两个 compression levels；三个 score families；总样本规模为 1,500 张测试图像，见 PAGE 4。因此结论不能外推为所有真实图像分布、所有生成器和所有 training-free detector 的绝对性能排序。尤其 per-generator cell 只有 100 张 fake，作者提醒 small differences between adjacent cells should not be over-read，见 PAGE 4。

第二，鲁棒性只测试 JPEG recompression，没有覆盖 resizing、blurring、social-network laundering pipelines 或 adversarial post-processing，见 PAGE 4。这一点对业务上线尤其重要，因为平台图片可能经历多种处理：缩放、裁剪、水印、截图、转发压缩、滤镜、超分或二次生成。本文证明 JPEG provenance 已足以改变 trend，但并没有证明其他 corruption 下的行为。

第三，RIGID-style 只使用 DINOv2-base，AEROBLADE-style 只使用一个 Stable Diffusion VAE，因此论文不能区分哪些现象是 DINOv2-base 或 `sd-vae-ft-mse` 特有，哪些是模型家族普遍属性，见 PAGE 4。对于 Midjourney 与 Wukong，由于架构不完全公开，Section 4 的机制性解释仍是 hypotheses，见 PAGE 4。

第四，本文没有研究 threshold transfer 或 calibration。所有结果都是 threshold-free AUROC，作者明确说明部署还需要额外研究 threshold transfer or calibration，见 PAGE 4。实际系统最终要选择阈值，且阈值会受业务误杀率、召回率、生成器分布和后处理链路影响。一个 AUROC 较高但校准不稳定的分数，仍可能不适合直接上线。

我的独立判断是，本文的最大价值在评测规范而非 detector ranking。由于样本规模小、只覆盖有限 corruption 和有限 score families，它不能告诉我们“哪个训练无关检测器最适合所有业务”；但它足以证明“只看单一 pooled AUROC 选择检测器”是不合格的评估流程。特别是 Table 3 与 Table 4 所示的方向反转，对任何基于阈值的上线系统都是高风险证据。

另一个独立判断是，本文未提供可确认公开代码会影响复现便利性。虽然论文报告了固定 seed、A100、Pillow JPEG re-encoding、DINOv2-base、Stable Diffusion VAE、LPIPS backbone、noise grid 等关键实现细节，见 PAGE 1-2，但没有公开仓库时，外部团队仍需自行重建数据管线。对于一篇强调 controlled audit 和 minimal reporting standards 的论文，公开评测脚本会显著增强其规范建议的可执行性。

## 结论

本文的贡献可以凝练为三条。第一，比较 training-free AI-generated image detectors 时，必须把 preprocessing 和 metric implementation 当作一等实验变量；LPIPS backbone 一项就带来 +0.085 AUROC，preprocessing 可让 per-generator ranking 改变最多约 0.38 AUROC，见 PAGE 2、PAGE 4。第二，score direction 必须按生成器族群和超参数验证；RIGID-style 在标准配置下可对 SD-family fakes 低于随机水平，且 naive fusion 会继承这种失败，见 PAGE 2-3。第三，压缩鲁棒性必须做 provenance-controlled re-encoding；否则“JPEG helps”这类反直觉结论可能来自 real/fake 格式偏差而不是检测器稳健性，见 PAGE 3-4。

对研究者而言，本文要求更细的报告粒度：per-generator、per-condition、score sign、preprocessing、backbone、encoding provenance 都应成为公开结果的一部分。对工程团队而言，本文给出的实用判断是：训练无关检测器可以作为候选组件，但不能跳过协议审计；在未验证目标生成器族群、转码链路和分数方向前，任何单一 AUROC 都不足以支持上线决策。

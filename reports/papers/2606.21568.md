## 背景与动机

学生行为检测（student behavior detection）是智能课堂分析（intelligent classroom analysis）中的核心视觉任务之一。其目标通常不是识别单个学生是否存在，而是在课堂图像或视频中定位多个学生实例，并判断其课堂行为类别，例如阅读、书写、抬头、睡觉、东张西望、低头和使用手机。论文摘要明确将该任务定位为智能课堂分析的重要组成部分，但同时指出大班场景中该问题仍然困难。（PAGE 证据不足：输入全文未含页码标记）

该任务与一般目标检测（general object detection）不同。一般检测任务虽然也处理多实例、多尺度和遮挡，但课堂行为检测还包含细粒度语义退化（fine-grained semantic degradation）：远距离学生目标往往面积很小，头部、手部、桌面和手机等关键视觉线索被压缩为少量像素，行为类别之间的局部差异容易消失。摘要明确列出 “dense instance co-occurrence, asymmetric occlusion, depth-wise scale variation, and fine-grained semantic degradation in distant targets” 四类挑战。（PAGE 证据不足：输入全文未含页码标记）

现有课堂行为数据集被论文认为不足以刻画这些困难。摘要称，已有课堂行为数据集和通用检测器不足以表征并解决上述问题。这意味着本文的动机不是单纯替换一个检测骨干，而是同时构建数据集与检测框架：数据集用于定义问题边界，模型用于对问题中的遮挡、密集和小目标结构作针对性建模。（PAGE 证据不足：输入全文未含页码标记）

本文提出 Highly Congested Classroom Behavior（HCCB）数据集。已知材料显示，该数据集包含 50,229 个学生行为实例，覆盖七个类别：reading、writing、heads up、sleeping、looking around、bowing head 和 using phone。这一类别设计集中在学生个体行为，而不是教师行为、课堂物体或互动事件。（PAGE 证据不足：输入全文未含页码标记）

在模型层面，本文提出 ODER-HSFNet，一种基于 YOLO 的检测框架。摘要将其描述为 “tailored to highly crowded classrooms”，即面向高度拥挤课堂场景定制。该框架包含三个任务特定模块：Occlusion-aware Deformable Edge Rectifier（ODER）、Hypergraph-State Spatial Fusion（HSSF）和 Occlusion-Calibrated Detection Head（OCDetect）。（PAGE 证据不足：输入全文未含页码标记）

OCDetect 的英文全称为 Occlusion-Calibrated Detection Head，可译为“遮挡校准检测头”。摘要称该模块用于 suppresses low-quality Pre-NMS candidates，并减少由 occlusion boundaries 和 neighboring instances 引起的 false positives。Pre-NMS candidates 指非极大值抑制（Non-Maximum Suppression, NMS）之前的候选框。（PAGE 证据不足：输入全文未含页码标记）

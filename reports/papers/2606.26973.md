一句话概括：本文把开放集半监督学习（Open-Set Semi-Supervised Learning, OSSL）中的样本筛选问题重写为梯度冲突控制问题，通过以监督梯度为锚点的几何投影，将有害的无标签辅助梯度修正到不反向破坏监督学习的安全区域内，从而在多数 CIFAR 与 ImageNet 开放集设置中提升闭集泛化和开放集鲁棒性（见 PAGE 1、PAGE 10、PAGE 12）。

## 背景与动机

半监督学习（Semi-Supervised Learning, SSL）的基本目标是在少量有标签数据与大量无标签数据之间建立有效的训练机制。标准 SSL 通常假设无标签数据来自同一组闭集类别，即无标签样本仍属于训练时定义的 ID 类别集合。论文指出，这一闭集假设在真实部署中很难成立，因为无标签数据流经常混入分布外样本（Out-of-Distribution outliers, OOD）（见 PAGE 1）。

开放集半监督学习（Open-Set Semi-Supervised Learning, OSSL）正是处理这一问题的设定：模型既要利用无标签数据，又要避免 OOD 样本破坏 ID 类别的分类边界。论文将已有 OSSL 方法概括为两类：一类是 filtering-based methods，即检测并拒绝可疑样本；另一类是 utilization-based methods，即尽可能利用无标签数据，常见做法包括软权重或将异常样本归入统一 unknown 类（见 PAGE 2、PAGE 4）。

过滤范式的优势是保守，能减少明显 OOD 噪声进入训练；但在低标签场景中，困难 ID 样本与 OOD 样本的可分性很弱，严格过滤会把有价值但不确定的 ID 样本排除掉。论文把这种现象称为 feature starvation，即模型失去困难 ID 样本提供的特征覆盖（见 PAGE 2、PAGE 3）。利用范式的优势是保留更多无标签信号，但若把困难 ID 样本错误地推向 unknown 类或错误伪标签，则可能主动削弱真实 ID 流形。

Figure 1 是全文动机的关键图。**用途**：说明样本级 OSSL 决策带来的双重困境，并展示无标签梯度与监督方向之间的几何关系。**读图要点**：左图对比 OpenMatch 的 ID recall 与 FixMatch 的 OOD inclusion，指向鲁棒性与覆盖率之间的权衡；右图用余弦相似度分布显示 OOD 梯度多表现为近似正交噪声，而误分类困难 ID 的梯度更可能与监督方向相反。**支撑的判断**：论文据此认为，OSSL 的核心风险不是单纯包含 OOD，而是错误目标诱导的 destructive conflict（见 PAGE 2、PAGE 3）。

因此，本文的出发点不是设计更强的 OOD 检测器，而是控制最终进入优化器的无标签梯度。只要辅助梯度不会在一阶意义上反向破坏监督目标，模型就可以在不完全依赖样本筛选的情况下利用一部分模糊无标签信号。这个问题转化构成 GGR 的方法论基础（见 PAGE 3、PAGE 5）。

## 预备知识

论文设定中，有标签集合记为 $D_l = \{(x_i^l, y_i^l)\}_{i=1}^{N_l}$，其中 $x_i^l$ 是有标签输入，$y_i^l$ 是其 ID 类别标签，ID 类别数为 $K$；无标签集合记为 $D_u = \{x_i^u\}_{i=1}^{N_u}$，其中可能混入 OOD 样本。模型参数记为 $\theta$，监督损失记为 $L_s$，无标签辅助损失记为 $L_u$，无标签损失权重为 $\lambda_u$（见 PAGE 5）。

论文将基础 OSSL 方法写成如下组合目标：

$$
\min_{\theta} J(\theta) := L_s(\theta; D_l) + \lambda_u L_u(\theta; D_u)
\tag{1}
$$

这个公式的含义是：训练目标由有标签监督项和无标签辅助项共同构成；$\lambda_u$ 控制无标签目标对整体优化的影响强度。问题在于，当 $L_u$ 来自错误伪标签或错误 unknown 归类时，它对应的梯度不一定服务于 $L_s$（见 PAGE 5）。

在第 $t$ 次迭代中，监督梯度记为 $g_s := \nabla_\theta L_s(\theta_t)$，辅助梯度记为 $g_u := \nabla_\theta L_u(\theta_t)$。符号 $\langle \cdot,\cdot\rangle$ 表示欧氏内积，用来衡量两个梯度方向是否一致。若 $\langle g_s, g_u\rangle < 0$，说明无标签辅助梯度在当前参数点上与监督下降方向相冲突，可能削弱 ID 特征（见 PAGE 5）。

论文关注的是 first-order non-interference，即一阶不干扰：在被修正的参数坐标块内，实际应用的辅助梯度不应与监督梯度相反。这个要求被写为：

$$
\langle g_s, \tilde{g}_u\rangle \ge 0
\tag{2}
$$

其中 $\tilde{g}_u$ 表示修正后的辅助梯度。直观解释是：GGR 不要求无标签信号一定与监督目标完全同向，但至少不能在一阶近似下反向抵消监督学习进展（见 PAGE 5）。

开放集指标方面，论文既报告 seen classes 上的闭集准确率，也报告包含 seen 与 unseen 类测试集上的开放集 balanced accuracy（BA）。开放集评测中，所有 unseen 类被视为统一 unknown 类，模型选择则基于闭集验证准确率，以保证不同方法的模型选择标准一致（见 PAGE 9、PAGE 10）。

## 方法详解

### 创新一：将 OSSL 困境重写为梯度冲突问题

传统 OSSL 方法通常围绕样本级判断展开：过滤方法试图判断“这个无标签样本是否可信”，利用方法试图决定“这个样本应该如何被纳入训练”。本文认为这两类方法共享同一瓶颈：低标签条件下，OOD 样本与困难 ID 样本很难被可靠区分，样本级错误会进一步转化为优化级错误（见 PAGE 2、PAGE 4）。

GGR 的第一项贡献是把这一问题从分类决策层面转化为梯度几何层面。对于错误目标产生的 $g_u$，关键不是它来自哪个样本，而是它是否与 $g_s$ 相反。若相反，则无标签损失正在抵消监督损失希望建立的 ID 判别结构；若近似正交，则它可能只是引入方差；若同向，则可能携带可利用的表示学习信号（见 PAGE 2、PAGE 3）。

这一改写直接对应论文 Figure 1 的右图。**用途**：展示 OOD、hard ID 与 rectified 梯度在余弦相似度空间中的分布。**读图要点**：OOD 更多表现为正交噪声，hard ID 在被错误目标驱动时更可能产生 adversarial gradients，rectified 分布被推向 constructive space。**支撑的判断**：样本级 OOD 检测不是唯一入口，梯度级控制可以在 OOD/ID 决策不完美时降低破坏性更新（见 PAGE 2）。

### 创新二：以监督梯度为锚点的向量级修正 VLR

GGR 的默认修正器是 vector-level rectifier（VLR）。它以监督梯度 $g_s$ 为锚点，定义一个安全半空间：

$$
H_{\mathrm{safe}}(g_s) :=
\{d \in \mathbb{R}^D \mid \langle d, g_s\rangle \ge 0\}
\tag{3}
$$

这里 $D$ 表示被选定 surgery scope 中展开后的参数维度，$d$ 是候选辅助更新方向。这个公式的含义是：只要候选方向与监督梯度的内积非负，它就在当前坐标块中不反向破坏监督目标（见 PAGE 5）。

然后，GGR 将原始辅助梯度 $g_u$ 投影到该安全半空间：

$$
\tilde{g}_u =
\arg\min_{d \in \mathbb{R}^D}
\frac{1}{2}\|d-g_u\|_2^2
\quad \mathrm{s.t.}\quad
\langle d,g_s\rangle \ge 0
\tag{4}
$$

这个公式说明，GGR 不是任意重写无标签梯度，而是在所有安全方向中选择与原始 $g_u$ 欧氏距离最近的方向。因此它遵循 minimum intervention，即只移除必要的破坏性成分，尽量保留非冲突辅助信号（见 PAGE 5、PAGE 7）。

该投影有闭式解：

$$
\tilde{g}_u =
\begin{cases}
g_u - \dfrac{\langle g_u,g_s\rangle}{\|g_s\|_2^2}g_s,
& \text{if } \langle g_u,g_s\rangle < 0 \text{ and } \|g_s\|_2 > 0,\\
g_u,
& \text{otherwise.}
\end{cases}
\tag{5}
$$

这条公式是 GGR 的核心操作。当 $g_u$ 与 $g_s$ 冲突时，它删除 $g_u$ 中指向 $-g_s$ 的分量；当不存在冲突时，它不改变 $g_u$。换言之，VLR 保留与监督梯度正交的部分，因为这部分可能仍含有表示学习信号（见 PAGE 6）。

修正后的训练更新为：

$$
\theta_{t+1} \leftarrow \theta_t - \eta(g_s + \lambda_u \tilde{g}_u)
\tag{6}
$$

其中 $\eta$ 是学习率。这个公式表明 GGR 是 post-hoc gradient-space operation：它不改变 forward pass，不要求显式 OOD 检测，也不改写底层 OSSL 方法的伪标签或损失定义，只是在优化器更新前替换辅助梯度（见 PAGE 6）。

### 创新三：用子空间锚点稳定小批量监督梯度

VLR 使用当前 mini-batch 的 $g_s$ 作为锚点，但低标签场景中 $g_s$ 可能噪声很大。为此，论文提出 subspace-aware rectification，用最近若干监督下降方向构成低维正交基 $U=[u_1,\ldots,u_k]\in \mathbb{R}^{D\times k}$，其中 $k\le d$，$d$ 是子空间维度。该基张成监督子空间 $\mathcal{S}=\mathrm{span}(U)$（见 PAGE 6）。

第一种子空间修正是 orthogonal subspace rectification（OSR）：

$$
\tilde{g}^{\mathrm{orth}}_u = (I-UU^\top)g_u
\tag{7}
$$

这里 $I$ 是单位矩阵，$UU^\top$ 是到监督子空间的投影。该公式的含义是：OSR 直接移除辅助梯度在监督子空间中的全部分量，只保留正交补方向，因此更保守。如果当前 $g_s$ 位于 $\mathcal{S}$ 中，则修正后的辅助梯度与 $g_s$ 精确正交（见 PAGE 6、PAGE 7）。

第二种子空间修正是 conic subspace rectification（CSR）：

$$
\tilde{g}^{\mathrm{signed}}_u =
g_u - U\min(U^\top g_u,0)
\tag{8}
$$

其中 $\min(\cdot,0)$ 按元素作用。该公式表示：CSR 只删除在监督基方向上为负的坐标，保留非负对齐成分，因此比 OSR 更激进地利用辅助信号。论文指出，当 $g_s$ 位于 $\{u_i\}$ 的 conic hull 中时，CSR 满足 $\langle g_s,\tilde{g}^{\mathrm{signed}}_u\rangle\ge 0$；当 $g_s$ 只是近似由该锥表示时，CSR 应被理解为近似锚定启发式而非严格保证（见 PAGE 7）。

Algorithm 1 给出完整训练过程。**用途**：说明 GGR 如何作为优化插件插入基础 OSSL 训练。**读图要点**：每次迭代先计算 $L_s$ 与 $L_u$，分别获得 surgery scope $P$ 上的 $g_s$ 与 $g_u$；若启用子空间则更新 $U$；随后根据 VLR、OSR 或 CSR 修正 $g_u$；最后在 $P$ 上使用 $g_s+\lambda_u\tilde{g}_u$，而 $P$ 外仍使用原始组合梯度。**支撑的判断**：GGR 的作用位置是梯度聚合阶段，而不是样本选择或损失建模阶段（见 PAGE 8）。

### 理论性质：最小干预、一阶不干扰与 regret 为零

论文首先证明，VLR 是投影到安全半空间的唯一欧氏最优解，并给出 alignment identity：

$$
\langle g_s,\tilde{g}_u\rangle
=
\max(0,\langle g_s,g_u\rangle)
\ge 0
$$

该公式说明：如果原始辅助梯度已经与监督梯度同向，GGR 不改变其内积；如果原始辅助梯度相反，GGR 将其与监督梯度的内积修正为 0。这正是 first-order non-opposition 的数学表达（见 PAGE 7、PAGE 19、PAGE 20）。

在选定 surgery block 上，论文定义组合更新方向 $v_t := g_s+\lambda_u\tilde{g}_u$，并证明 $\langle g_s,v_t\rangle\ge\|g_s\|_2^2$。若监督损失关于该坐标块是 $L$-smooth 且学习率满足相应上界，则有：

$$
L_s(z_{t+1};\theta_{t,\setminus P})
\le
L_s(z_t;\theta_{t,\setminus P})
-
\frac{\eta}{2}\langle g_s,v_t\rangle
\tag{9}
$$

其中 $z$ 是 surgery scope $P$ 内参数展开后的向量，$\theta_{t,\setminus P}$ 表示 $P$ 外坐标保持固定。这个公式只给出局部、分块、一阶条件下的监督损失下降保证；它不宣称完整目标 $L_s+\lambda_uL_u$ 单调下降（见 PAGE 7、PAGE 8、PAGE 20、PAGE 21）。

论文还定义 applied conflict regret：

$$
R_T^{\mathrm{app}}
=
\sum_{t=1}^{T}
\max(0,-\langle g_s^{(t)}, g_{\mathrm{app}}^{(t)}\rangle)
$$

其中 $g_{\mathrm{app}}^{(t)}=\lambda_u\tilde{g}_u^{(t)}$ 是真正进入优化器的加权辅助更新。对 VLR 而言，该 regret 在精确算术下为 0，因为每一步修正后都满足非负内积（见 PAGE 8、PAGE 22、PAGE 27）。

### 与 PCGrad 的差异

GGR 与 PCGrad 都涉及梯度投影，但目标不同。PCGrad 来自多任务学习，通常对任务梯度作对称修正；GGR 面向 OSSL，监督梯度来自真实标签，辅助梯度可能来自错误伪标签或开放集目标，因此采用 asymmetric anchor-preserving projection：固定 $g_s$，只修正 $g_u$（见 PAGE 4、PAGE 25）。

论文在 Appendix A.5 中给出对比。若 $c=\langle g_s,g_u\rangle<0$，GGR 的监督进展满足：

$$
\langle g_s,v_{\mathrm{ggr}}\rangle=\|g_s\|_2^2
$$

而对称 PCGrad 聚合的监督进展为：

$$
\langle g_s,v_{\mathrm{pcgrad}}\rangle
=
\|g_s\|_2^2
\left(1-\cos^2(g_s,g_u)\right)
$$

这个结果说明，当 $g_s$ 与 $g_u$ 近乎相反时，对称 PCGrad 可能显著削弱监督锚点的进展；GGR 则保留监督梯度本身，只删除辅助梯度中破坏监督方向的分量（见 PAGE 25）。

### 子空间修正与诊断实验

Figure 2 分析 OSR 与 CSR 在 CIFAR-100 50/50 split 下不同子空间维度 $d\in\{1,5,10,20,30,40,50\}$ 的表现。**用途**：比较子空间锚点相对 VLR 的稳定性与利用率。**读图要点**：OSR 更保守，开放集 BA 在多个维度上更稳定；CSR 保留更多正向辅助分量，可能获得更高闭集峰值，但对基估计质量更敏感。**支撑的判断**：中等 $d$ 能在稳定锚点与不过度压制无标签信号之间取得平衡，过小锚点不足，过大则可能抑制有益辅助信号（见 PAGE 12、PAGE 13）。

Figure 3 是梯度冲突诊断。**用途**：验证 GGR 是否真的降低“进入优化器的冲突”。**读图要点**：raw conflict rate 在 GGR 下仍可能存在，因为底层损失和伪标签机制没有改变；final conflict rate 接近 0，因为 GGR 在更新前修正了辅助梯度；cumulative conflict regret 基线显著累积，而 GGR 近乎平坦。**支撑的判断**：GGR 不消除损失层面的风险源，而是在优化执行层面阻断 destructive update（见 PAGE 13、PAGE 14、PAGE 27）。

## 讨论

GGR 的适用边界首先来自其插件化假设。它适合那些训练过程可以显式分离监督损失梯度 $g_s$ 与无标签辅助损失梯度 $g_u$ 的 OSSL 框架；若某些方法的监督与辅助目标高度耦合、难以在实现中分离梯度，则需要额外工程改造。论文的 Algorithm 1 与 implementation details 均假设可以在 surgery scope 上分别计算并重组梯度（见 PAGE 8、PAGE 11、PAGE 26）。

从业务角度看，GGR 对自动标注、半监督数据利用、伪标签噪声控制具有直接相关性。若未标注数据中混入脏样本，传统流程通常需要更强数据清洗或更严格阈值；GGR 提供另一条路径，即允许模糊样本进入辅助目标，但在梯度级别阻止其反向破坏监督 ID 学习。该推论与论文的分类实验一致，但论文没有在检测、ReID 或大规模业务数据上验证，因此不能直接外推为生产收益（见 PAGE 9 至 PAGE 15）。

方法论上，GGR 的关键价值在于把“样本是否可信”的离散决策转化为“更新方向是否安全”的连续几何约束。这样做降低了对完美 OOD 检测的依赖，同时保留一部分困难 ID 与近似正交 OOD 可能带来的表示学习信号。不过，它并不替代 OOD modeling、uncertainty estimation 或更可靠伪标签机制；论文结论也明确称其与现有 detection heuristics 互补（见 PAGE 15）。

## 局限分析

作者自述的第一类局限是理论保证的局部性。论文第 6 节明确指出，结果是 local and first-order：它们只表明当前迭代、当前 surgery coordinate block 内的辅助更新不反向监督下降方向，并不推出全局最优，也不保证完整组合目标 $L_s+\lambda_uL_u$ 单调下降（见 PAGE 14、PAGE 15）。

作者自述的第二类局限是锚点质量。若 mini-batch 监督梯度本身 highly noisy or uninformative，VLR 的瞬时锚点可能过于保守或不稳定；OSR 与 CSR 通过时间平滑部分缓解这一问题，但引入子空间维度 $d$ 后又产生额外超参数与计算开销（见 PAGE 14、PAGE 15）。

本文的独立判断是：实验范围仍以图像分类 OSSL benchmark 为主，虽然覆盖 CIFAR 与 ImageNet-30，但没有证明在目标检测、行人/物体 ReID、多标签属性识别或真实大规模未标注业务数据上的收益。因此，对“自动标注/数据”方向的价值应理解为可验证假设，而不是已被论文证明的结论（见 PAGE 9 至 PAGE 15）。

另一项独立判断是：代码可复现性证据不足。论文第 1 页称 “Code will be available” 并给出 GitHub 链接，但当前可确认材料中没有核心源码、配置文件或训练脚本；因此无法完成论文方法与源码函数的逐项对应，也不能判断分离梯度计算、surgery scope 展开、混合精度下数值误差等实现细节是否与论文描述完全一致（见 PAGE 1）。

此外，用户提供的 `figures` 列表为空，因此本文无法嵌入论文图像，只能依据文本抽取中的 Figure 1、Figure 2、Figure 3 描述进行解释。图像级证据不足不影响主要数值表格和公式分析，但会限制对图中曲线细节、坐标刻度和视觉分布形态的独立复核。

## 结论

本文提出的 Geometric Gradient Rectification（GGR）将 OSSL 中的开放集样本噪声问题转化为梯度几何控制问题。它以监督梯度为受保护锚点，通过 VLR、OSR、CSR 等投影修正辅助梯度，使实际应用的无标签更新在选定坐标块内满足一阶非反对约束。理论上，VLR 是对安全半空间的最小欧氏投影，并带来 applied conflict regret 为零的性质；经验上，GGR 在 CIFAR-10/100 与 ImageNet-30 的多数设置中提高 OpenMatch、IOMatch、DAC 等代表性 OSSL 方法的闭集与开放集表现（见 PAGE 5 至 PAGE 15）。

更重要的是，GGR 给出了一种可迁移的研究视角：当样本级 OOD/ID 判断不可靠时，可以在优化器之前对辅助信号做几何审查。对于自动标注和脏无标签数据利用场景，这一思想有潜在价值，但仍需要在检测、ReID、属性模型和真实业务数据规模上做小规模验证后再判断工程收益。
